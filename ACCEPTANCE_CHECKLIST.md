## 验收标准说明

本文档列出了 CodeMap 编排层重构项目的完整验收测试用例。所有测试用例必须通过后方可视为重构完成。

**测试用例设计原则：**
1. **覆盖所有功能点**：每个设计文档中的功能都有对应的测试用例
2. **覆盖所有边界条件**：空输入、错误输入、极端情况
3. **覆盖所有集成点**：模块间交互、工具链调用
4. **覆盖性能指标**：响应时间、Token 限制
5. **覆盖兼容性**：向后兼容、跨平台

## 一、核心功能测试 (Core Functionality)

### 1.1 CLI 命令测试

#### TC-CLI-001: analyze 命令基础执行
**测试内容**: 验证 `codemap analyze` 命令可正常执行
**前置条件**: 项目已构建，CLI 可调用
**测试步骤**:
1. 执行 `codemap analyze --intent search --keywords parser`
2. 观察命令输出
**预期结果**:
- 命令正常执行，不抛出异常
- 返回搜索结果列表
- 输出包含置信度信息
**为什么这样设计**: 这是主入口命令，必须确保基础可用性
**优先级**: P0 | **类型**: 功能

---

#### TC-CLI-002: analyze 命令参数验证 - 缺少必需参数

1. 执行 `codemap analyze` (不带任何参数)

**预期结果**:
- 返回清晰的错误信息

**为什么这样设计**: 确保用户获得明确的参数错误提示
**优先级**: P1 | **类型**: 边界

---

#### TC-CLI-003: analyze 命令 JSON 输出格式
**测试内容**: 验证 `--json` 参数输出格式
**测试步骤**:

**预期结果**:
- 输出为有效 JSON 格式

**为什么这样设计**: JSON 输出供程序调用，格式必须稳定
**优先级**: P1 | **类型**: 功能

## 二、置信度机制测试 (Confidence System)

### 2.1 置信度计算测试

#### TC-CONF-001: 高置信度场景识别
**测试内容**: 验证高置信度场景正确识别
**测试步骤**:
1. 搜索常见关键词 "parser"（预期返回多个高相关结果）
2. 检查置信度输出
**预期结果**:
- 置信度 score >= 0.6 (search intent 的高阈值)
- level = 'high'
- reasons 包含结果数量和最高相关度
**为什么这样设计**: 确保高质量结果不被错误地触发回退
**优先级**: P0 | **类型**: 功能

---

#### TC-CONF-002: 低置信度触发回退
**测试内容**: 验证低置信度时触发回退机制
**测试步骤**:
1. 搜索罕见术语 "xyznotexist123"（预期返回极少结果）
2. 观察置信度
**预期结果**:
- 初始置信度 < 0.25 (search intent 的中阈值)
- 不再触发外部 rg 回退（rg-internal 仅内部调试用）
- 输出包含置信度提示
**为什么这样设计**: 核心功能 - 低置信度时返回提示而非回退
**优先级**: P0 | **类型**: 功能

---

#### TC-CONF-003: 不同 Intent 的阈值差异
**测试内容**: 验证不同 intent 使用不同阈值
**测试步骤**:
1. 使用相同低质量输入测试不同 intent:
   - `impact`: 阈值 high=0.7, medium=0.4
   - `search`: 阈值 high=0.5, medium=0.25
2. 比较触发回退的时机
**预期结果**:
- impact intent 更早触发回退（阈值更高）
- search intent 容忍度更高
**为什么这样设计**: 结构化数据分析需要更高置信度
**优先级**: P1 | **类型**: 功能

---

#### TC-CONF-004: 置信度计算准确性
**测试内容**: 验证置信度计算公式正确性
**测试步骤**:
1. 构造已知数量的搜索结果（如 5 个结果，平均相关度 0.8）
2. 计算预期置信度:
   - countScore = min(5/5, 1) * 0.4 = 0.4
   - qualityScore = 0.8 * 0.4 = 0.32
   - scenarioScore = 0.2 (如果 top > 0.9) 或 0
3. 比较实际输出
**预期结果**:
- 实际置信度与计算值一致
- reasons 数组内容与评分逻辑匹配
**为什么这样设计**: 置信度是核心算法，计算必须准确
**优先级**: P0 | **类型**: 算法

---

#### TC-CONF-005: 空结果置信度
**测试内容**: 验证无结果时的置信度计算
**测试步骤**:
1. 搜索不存在的关键词
2. 检查置信度输出
**预期结果**:
- 置信度 = 0
- level = 'low'
- reasons 包含 "无搜索结果"
**为什么这样设计**: 边界条件必须正确处理
**优先级**: P1 | **类型**: 边界
