# Triad 工作流定义

## 概述

本文档定义工作流编排器测试任务的Triad协作流程，确保高质量交付。

## 工作流阶段

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  generator  │────▶│     qa      │────▶│ supervisor  │
│   (生成)     │     │   (审查)     │     │   (验收)     │
└─────────────┘     └─────────────┘     └─────────────┘
       ▲                    │                    │
       └────────────────────┴────────────────────┘
                    (反馈循环)
```

## Phase 1: 生成阶段 (Generator)

### 输入
- 源代码文件
- 任务需求
- 技术约束

### 活动
1. **分析源代码**
   - 读取并理解5个源文件
   - 识别公共方法和类型
   - 分析依赖关系

2. **生成任务四件套**
   - PROMPT.md: 完整的需求描述
   - EVAL.ts: 可执行的检查点
   - SCORING.md: 评分标准
   - task-metadata.yaml: 任务元数据

3. **生成测试代码**
   - 5个测试文件
   - 使用Vitest
   - 100%覆盖率
   - 完整的Mock

### 输出
- 任务四件套文件
- 5个测试文件
- 生成报告

### 完成标准
- [ ] 所有文件已创建
- [ ] 代码结构符合要求
- [ ] 准备提交QA审查

## Phase 2: 审查阶段 (QA)

### 输入
- 生成的测试代码
- 任务四件套
- 源代码

### 活动
1. **覆盖率验证**
   - 运行测试套件
   - 检查覆盖率报告
   - 验证达到100%

2. **边界条件检查**
   - null/undefined处理
   - 空值处理
   - 错误路径覆盖

3. **Mock验证**
   - fs模块mock正确
   - 依赖类mock正确
   - 无真实IO

4. **代码质量检查**
   - 无any类型
   - 描述清晰
   - 结构合理

### 输出
- QA审查报告
- 问题列表（如有）
- 改进建议

### 完成标准
- [ ] 覆盖率100%验证
- [ ] 边界条件完整
- [ ] Mock使用正确
- [ ] 代码质量合格

### 决策点
- **通过** → 提交Supervisor
- **需修改** → 反馈Generator

## Phase 3: 验收阶段 (Supervisor)

### 输入
- QA审查报告
- 测试代码
- 覆盖率报告

### 活动
1. **运行质量门禁**
   - 执行EVAL.ts检查点
   - 验证评分标准
   - 检查关键路径

2. **最终验收**
   - 审查QA报告
   - 验证完成标准
   - 做出决策

### 输出
- 验收报告
- 合并批准/拒绝
- 完成记录

### 决策点
- **批准(A级)** → 合并
- **有条件批准(B级)** → 合并，记录改进
- **需修改(C级)** → 返回QA
- **拒绝(D级)** → 返回Generator重做

## 反馈循环

### Generator → QA
```
1. Generator完成任务
2. 提交到任务队列
3. QA自动分配审查
4. QA完成审查并反馈
```

### QA → Generator (修复)
```
1. QA发现问题
2. 创建详细反馈
3. Generator接收反馈
4. Generator修复问题
5. 重新提交QA审查
```

### QA → Supervisor
```
1. QA审查通过
2. 提交验收请求
3. Supervisor分配审查
4. 完成验收决策
```

## 时间限制

| 阶段 | 建议时间 | 最大时间 |
|------|----------|----------|
| 生成 | 30分钟 | 60分钟 |
| 审查 | 15分钟 | 30分钟 |
| 验收 | 10分钟 | 20分钟 |

## 升级路径

如果某个阶段超时或遇到阻塞：
1. 记录阻塞原因
2. 升级给上级角色
3. 协商解决方案
4. 记录决策

## 完成定义 (Definition of Done)

- [ ] 5个测试文件存在
- [ ] 所有测试通过
- [ ] 覆盖率100%
- [ ] QA审查通过
- [ ] Supervisor批准
- [ ] 已合并到主分支
