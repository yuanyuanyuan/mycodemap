# 工作流编排器测试评分标准

## 总分：100分

---

## 评分维度

### 1. 测试完整性 (40分)

| 检查项 | 分值 | 说明 |
|--------|------|------|
| 所有公共方法有测试 | 20分 | 每个导出方法至少一个测试用例 |
| 所有状态转换覆盖 | 20分 | 覆盖完整状态机: pending → running → completed → verified |

**评分细则：**
- 20分：所有方法/状态都有测试
- 15分：遗漏1-2个次要方法/状态
- 10分：遗漏3-5个
- 5分：遗漏超过5个
- 0分：大量遗漏

---

### 2. Mock正确使用 (25分)

| 检查项 | 分值 | 说明 |
|--------|------|------|
| fs模块正确mock | 10分 | 使用 vi.mock('node:fs')，正确模拟readFile/writeFile等 |
| 依赖类正确mock | 10分 | ToolOrchestrator等依赖使用vi.mock模拟 |
| vi.fn() spy正确使用 | 5分 | 正确使用spy验证函数调用 |

**评分细则：**
- 10分/5分：完全正确使用
- 7分/4分：少量问题
- 5分/3分：部分问题
- 3分/2分：较多问题
- 0分：完全错误或未使用

---

### 3. 边界条件覆盖 (20分)

| 检查项 | 分值 | 说明 |
|--------|------|------|
| 异常输入处理 | 10分 | null/undefined/空值处理 |
| 边界处理 | 10分 | 空Map/Set、文件不存在、无效状态转换 |

**边界条件清单：**
- [ ] null参数传入
- [ ] undefined参数传入
- [ ] 空字符串处理
- [ ] 空Map序列化
- [ ] 空Set序列化
- [ ] 文件不存在时返回null
- [ ] 无效状态转换返回false
- [ ] 强制推进模式(force=true)

---

### 4. 代码质量 (15分)

| 检查项 | 分值 | 说明 |
|--------|------|------|
| TypeScript类型正确 | 5分 | 无any类型，类型导入正确 |
| 测试描述清晰 | 5分 | describe/it描述准确反映测试内容 |
| 无冗余代码 | 5分 | 无重复代码，结构清晰 |

---

## 评分等级

| 等级 | 分数范围 | 说明 | 后续动作 |
|------|----------|------|----------|
| **A** | 90-100 | 优秀，完全符合要求 | 直接合并 |
| **B** | 75-89 | 良好，核心功能覆盖 | 合并，记录小改进点 |
| **C** | 60-74 | 及格，基本功能覆盖 | 需补充部分测试后合并 |
| **D** | < 60 | 不及格 | 需重写 |

---

## 快速评分检查表

### 必须项 (任一不通过直接降级到C或D)

- [ ] 5个测试文件全部存在
- [ ] 使用 vi.mock('node:fs')
- [ ] 所有测试通过 (npm test)
- [ ] 无TypeScript编译错误

### 加分项

- [ ] 覆盖率100% (+5分)
- [ ] 有详细的测试注释 (+3分)
- [ ] 使用测试工厂函数 (+2分)
- [ ] 有性能测试 (+3分)

### 扣分项

- [ ] 测试之间相互依赖 (-10分)
- [ ] Mock未清理导致状态泄漏 (-10分)
- [ ] 使用真实文件系统 (-20分)
- [ ] 存在 `any` 类型 (-5分/个)
- [ ] 测试描述不清晰 (-3分/个)

## 评分工具

运行以下命令进行自动评分：

```bash
# 运行测试
npm test -- src/orchestrator/workflow/__tests__

# 检查覆盖率
npm run test:coverage -- src/orchestrator/workflow/__tests__

# TypeScript检查
npx tsc --noEmit
```
