# 三角色工作流 - 适配器测试套件生成

## 工作流概览

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  Generator  │───→│     QA      │───→│  Supervisor │
│  (生成器)    │    │  (审查员)    │    │   (主管)     │
└─────────────┘    └─────────────┘    └─────────────┘
      │                   │                   │
      ▼                   ▼                   ▼
   生成测试代码         质量审查            最终验收
```

## Phase 1: 需求解析 (Generator)

**输入**: 设计文档、接口定义、源代码结构

**输出**: 
- 测试策略文档
- 测试范围界定
- 边界条件清单

**关键活动**:
1. 分析ToolAdapter接口契约
2. 识别AstGrepAdapter公共方法
3. 识别CodemapAdapter功能点
4. 定义边界条件矩阵

## Phase 2: 测试设计 (Generator)

**设计原则**:
- AAA模式 (Arrange-Act-Assert)
- 独立性原则
- 可重复性原则
- 快速反馈原则

**输出**:
- 测试用例列表
- Mock策略设计
- 测试数据结构

## Phase 3: 代码生成 (Generator)

**交付物**:
- `base-adapter.test.ts` - 接口契约测试
- `ast-grep-adapter.test.ts` - 功能测试
- `codemap-adapter.test.ts` - 结构分析测试

**代码规范**:
```typescript
// 标准测试结构
describe('ModuleName', () => {
  describe('methodName', () => {
    it('should do something when condition', () => {
      // Arrange
      // Act
      // Assert
    });
  });
});
```

## Phase 4: 质量审查 (QA)

### 检查点清单

#### 4.1 覆盖率检查
- [ ] 语句覆盖率 = 100%
- [ ] 分支覆盖率 = 100%
- [ ] 函数覆盖率 = 100%
- [ ] 行覆盖率 = 100%

#### 4.2 边界条件检查
- [ ] 空输入处理
- [ ] 极大值输入
- [ ] 异常类型测试
- [ ] 异步错误处理

#### 4.3 Mock验证
- [ ] 外部依赖已隔离
- [ ] Mock返回值合理
- [ ] 模拟了错误场景

#### 4.4 代码质量
- [ ] 无重复测试逻辑
- [ ] 测试名称清晰
- [ ] 断言明确具体

## Phase 5: 验收决策 (Supervisor)

### 验收标准

| 维度 | 权重 | 通过标准 |
|------|------|----------|
| 功能覆盖 | 40% | 100%方法覆盖 |
| 边界条件 | 30% | 所有边界已测试 |
| 代码质量 | 20% | 无代码异味 |
| 文档完整 | 10% | 测试意图清晰 |

### 决策矩阵

| 总分 | 决策 |
|------|------|
| 90-100 | 通过，优秀 |
| 80-89 | 通过，需改进 |
| 70-79 | 条件通过，必须修复 |
| <70 | 拒绝，重新生成 |

## 交接规则

### Generator → QA
必须满足：
1. 所有测试可执行
2. 无TypeScript类型错误
3. 附带初始覆盖率报告

### QA → Supervisor
必须满足：
1. 所有检查点已验证
2. 问题清单已清空或已记录
3. 覆盖率达到100%

## 异常处理

### 覆盖率不达标
**处理方式**: 退回Generator，补充测试用例

### 边界条件缺失
**处理方式**: 退回Generator，添加边界测试

### 架构设计问题
**处理方式**: 升级至Supervisor，重新设计
