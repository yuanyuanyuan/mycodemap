## 验收流程

### Step 1: 自动化检查
```bash
# 运行测试
npm test

# 检查覆盖率
npm run test:coverage

# 类型检查
npx tsc --noEmit
```

### Step 2: 人工审查
1. QA执行检查点验证
2. 标记未通过项
3. 生成分数报告

### Step 3: 决策会议
- Supervisor主持
- Generator和QA参与
- 讨论未通过项
- 做出最终决策

### Step 4: 交付确认
- 签署验收报告
- 归档测试代码
- 关闭任务

## 强制拒绝条件

以下任何一项触发自动拒绝：
- [ ] 覆盖率低于100%
- [ ] 测试无法运行
- [ ] 存在TypeScript类型错误
- [ ] 缺少错误处理测试
- [ ] 关键边界条件未覆盖
