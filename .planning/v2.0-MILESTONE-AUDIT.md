## Critical Blockers (4)

### 1. Global Flags Orphaned (TRUST-04, INST-01/02)

`--wasm-fallback` and `--apply-suggestion` are registered as global flags but **never consumed** by any command action. `tryApplySuggestion()` and `checkAndActivateWasmFallback()` exist but have **zero call sites** outside their own test files.

**Fix:** Wire these into the command error handling pipeline in `index.ts` or individual command handlers.

### 2. validateTreeSitter Blocks WASM Fallback (INST-01, TRUST-02)

`index.ts:77-88` calls `validateTreeSitter()` which can throw before `loadTreeSitter()` (Phase 47's WASM-aware loader) is ever invoked. The WASM fallback path is architecturally unreachable.

**Fix:** Make `validateTreeSitter` use the loader-aware check, or remove the validation and let the loader handle fallback.

### 3. tryApplySuggestion Never Called (TRUST-04)

The auto-remediation engine (`tryApplySuggestion`) is fully implemented with 3-gate safety but **never invoked**. The `--apply-suggestion` flag is parsed but `cliOpts.applySuggestion` is never read.

**Fix:** Call `tryApplySuggestion` from the error handling path in command action handlers.

### 4. Contract Schema Coverage Incomplete (AGENT-06)

Only `analyze`, `query`, `deps` are in the contract schema. The MCP gateway auto-registration only works for these 3 commands. `doctor`, `benchmark`, `init`, `generate`, and 10+ others are NOT auto-registered.

**Fix:** Add remaining commands to contract schema, or accept as progressive migration backlog.

## Warnings (5)

1. **benchmark bypasses shared output infrastructure** — Uses `console.log` directly
2. **Contract missing global flags** — `--wasm-fallback`, `--apply-suggestion`, `--native` not in schema
3. **Type mismatch** — Contract declares `number` defaults but Commander uses `string` defaults
4. **outputSchema mismatch** — MCP `outputSchema` describes `cli_redirect`, not contract `outputShape`
5. **Zod default/optional ordering** — Produces required properties for optional flags with defaults
