</read_first>

<action>
Create `src/cli/env-contract/types.ts` with explicit exported unions:

```typescript
export type ContractCategory = 'execution' | 'commit' | 'retrieval' | 'validation' | 'style';
export type ContractSeverity = 'critical' | 'high' | 'medium' | 'low';
export type SourceAuthority = 'executable' | 'governance' | 'generated' | 'example';
export type AgentType = 'explore' | 'plan' | 'edit' | 'worker' | 'review' | 'verify' | 'default';
```

Create interfaces:

```typescript
export interface ContractSource {
  file: string;
  line?: number;
  hash: string;
  authority: SourceAuthority;
}

export interface ContractItem {
  id: string;
  category: ContractCategory;
  severity: ContractSeverity;
  content: string;
  metadata?: Record<string, unknown>;
  sources: ContractSource[];
}

export interface ContractConflict {
  id: string;
  severity: ContractSeverity;
  description: string;
  sources: Array<{ file: string; value: string }>;
  recommendation: string;
}

export interface ProjectEnvironmentContract {
  schemaVersion: 'env-contract.v1';
  generatedAt: string;
  projectProfile: {
    name: string;
    source: string;
    confidence: 'high' | 'medium' | 'low' | 'none';
  };
  items: ContractItem[];
  conflicts: ContractConflict[];
  sourceSnapshots: Array<{ file: string; hash: string; lastModified: string }>;
}
```

Create `src/cli/env-contract/validation.ts` with `validateContractItem()` and `validateProjectEnvironmentContract()` that enforce:
- categories are exactly `execution|commit|retrieval|validation|style`
- severities are exactly `critical|high|medium|low`
- `content` is non-empty
- every item has at least one source
- every source has `file`, `hash`, and `authority`
- `schemaVersion` is exactly `env-contract.v1`

Create `src/cli/env-contract/index.ts` exporting the public types and functions added in this plan.
</action>

</read_first>

<action>
Create `src/cli/env-contract/discovery.ts` with a main function:

```typescript
export function discoverProjectEnvironmentContract(rootDir: string, options?: {
  profileName?: string;
  generatedAt?: string;
}): ProjectEnvironmentContract
```

The discovery function must create these critical items when the source facts exist:
- `shell-rtk-wrapper`: category `execution`, severity `critical`, content says shell commands must use `rtk`, source from `AGENTS.md` or relevant docs.
- `commit-format`: category `commit`, severity `critical`, content says commit messages use `[TAG] scope: message`, metadata contains valid tags from `.githooks/commit-msg`.
- `test-entry-vitest`: category `execution`, severity `critical`, content says tests run with `npx vitest run` / package script `test: vitest run`.
- `codemap-query-priority`: category `retrieval`, severity `high`, content says CodeMap CLI query/analyze/deps/impact should be tried before raw grep/rg for code search.
- `real-scenario-validation`: category `validation`, severity `high`, content says real filesystem/subprocess or transport evidence and at least one failure scenario are required.

Record `sourceSnapshots` for every source file read. Each snapshot must include a sha256 hash and last modified timestamp.

Use the source authority order:
- `.githooks/*`, `package.json`, `vitest*.config.ts` -> `executable`
- `AGENTS.md`, `docs/rules/*.md`, `.mycodemap/rules/*.md` -> `governance`
- `.mycodemap/assistants/*` -> `generated`
- examples/archive docs -> `example`
</action>
