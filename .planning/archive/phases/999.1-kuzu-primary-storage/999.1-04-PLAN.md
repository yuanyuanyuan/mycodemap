    # CodeMap 存储配置

    CodeMap 支持多种存储后端，可根据项目需求选择。

    ## 存储类型

    ### KùzuDB（推荐）

    嵌入式图数据库，适合中大型项目。

    ```json
    {
      "storage": {
        "type": "kuzudb",
        "databasePath": ".codemap/kuzu"
      }
    }
    ```

    **⚠️ 重要提示**: KùzuDB 项目已于 2025-10-10 归档。详见 [风险评估](../src/infrastructure/storage/RISK_ASSESSMENT.md)。

    **特性**:
    - 原生图模型（节点/关系表）
    - Cypher 查询语言
    - 高性能复杂依赖分析
    - 自动 fallback 到文件系统

    ### 文件系统

    JSON 文件存储，适合小型项目或作为 fallback。

    ```json
    {
      "storage": {
        "type": "filesystem",
        "outputPath": ".codemap/storage"
      }
    }
    ```

    **特性**:
    - 零依赖，开箱即用
    - 人类可读的 JSON 格式
    - 适合版本控制

    ### Auto 模式

    自动选择存储类型（优先 KùzuDB，失败时 fallback 到文件系统）。

    ```json
    {
      "storage": {
        "type": "auto",
        "autoThresholds": {
          "useGraphDBWhenFileCount": 1000,
          "useGraphDBWhenNodeCount": 10000
        }
      }
    }
    ```

    ## 数据迁移

    从文件系统迁移到 KùzuDB 是自动的：

    1. 配置 `type: "kuzudb"` 或 `type: "auto"`
    2. CodeMap 会自动检测文件系统数据
    3. 自动迁移到 KùzuDB
    4. 原文件自动备份为 `codegraph.json.backup.{timestamp}`

    ## Fallback 机制

    当 KùzuDB 不可用时（如未安装、平台不兼容），CodeMap 会自动降级到文件系统：

    ```
    [StorageFactory] KùzuDB unavailable, falling back to filesystem
    ```

    ## 配置示例

    ### 小型项目（< 100 文件）

    ```json
    {
      "storage": {
        "type": "filesystem"
      }
    }
    ```

    ### 中型项目（100-1000 文件）

    ```json
    {
      "storage": {
        "type": "auto"
      }
    }
    ```

    ### 大型项目（> 1000 文件）

    ```json
    {
      "storage": {
        "type": "kuzudb",
        "databasePath": ".codemap/kuzu"
      }
    }
    ```

    ## 故障排除

    ### KùzuDB 安装失败

    ```bash
    npm install kuzu
    ```

    如果安装失败，CodeMap 会自动使用文件系统。

    ### 数据迁移失败

    检查日志输出，手动迁移：

    ```bash
    # 备份现有数据
    cp .codemap/storage/codegraph.json .codemap/storage/codegraph.json.manual-backup

    # 删除 KùzuDB 数据库（如果损坏）
    rm -rf .codemap/kuzudb

    # 重新运行分析
    codemap analyze
    ```
