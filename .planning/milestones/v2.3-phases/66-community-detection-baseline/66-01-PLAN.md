<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted graph truth → community projection | 混合 module/symbol dependency 必须被正确折叠，不能假造不存在的 edge kinds |
| projection weights → Louvain partition | 错误权重或 synthetic kinds 会篡改社区边界，导致模块边界解释失真 |
| shared community result → MCP adapter | MCP 不能重新发明第二套语义或丢失 degraded-state warnings |
| sparse graph reality → downstream agents | 弱信号 partition 不能被包装成高置信结构真相 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-66-01 | T | community projection weight mapping | mitigate | 仅允许 `call|inherit|implement|import|type-ref` 映射到显式常量权重；缺失的 `TESTED_BY` / `CONTAINS` 不得 synthetic 注入，测试锁定 real-kind 映射 |
| T-66-02 | I | low-signal community interpretation | mitigate | 在 shared helper 内计算 sparse / singleton-heavy / dominant-cluster heuristics，并把 `confidence='reduced'` + warning codes 写入结果与 MCP payload |
| T-66-03 | I | missing or partial graph truth | mitigate | 复用 Phase 65 的 `GRAPH_NOT_FOUND` / `GRAPH_PARTIAL` 语义；无 graph 时返回 `unavailable`，partial graph 时保留 warnings 与 remediation |
| T-66-04 | D | MCP payload size / noisy output | mitigate | community 输出固定按 `size` 排序，并限制 `topPaths` 最多 5 条、`dominantEdgeKinds` 最多 3 条，避免巨大 payload 或噪声淹没 agent |
| T-66-05 | R | doc / runtime drift on new MCP tool | mitigate | 同步更新 `AI_GUIDE.md` 与 `ARCHITECTURE.md`，并用 `CodeMapMcpServer.test.ts` 与 real MCP e2e 证明 `codemap_communities` 已按文档注册和返回 |
</threat_model>
