<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI/MCP input → entrypoint resolver | file path 与 symbol name 都可能缺失、歧义或越界 |
| entrypoint resolver → traversal engine | resolver 不能把错误或歧义静默缩成单个候选 |
| graph metadata → surface confidence/warnings | partial/stale graph 不得伪装成完整 truth |
| shared contract → public surfaces | CLI/MCP 不能各自再塑造第二份 impact 语义 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-65-01 | T | entrypoint resolver | mitigate | unresolved / ambiguous file/symbol 都返回显式 structured status 与 remediation，禁止 silent narrowing |
| T-65-02 | I | direct/transitive layering | mitigate | layer partition 作为 shared contract truth 在 helper/storage 层落地，并用 tests 锁住 hop 分组与 representative path |
| T-65-03 | I | degraded graph semantics | mitigate | `partial` / `missing` / `truncated` 必须映射成 warning 或 reduced confidence，而非高置信空成功 |
| T-65-04 | R | CLI/MCP drift | mitigate | CLI/MCP 只消费 shared contract；任何 shape change 都在 types + tests + docs 同步 |
| T-65-05 | E | HTTP scope creep | accept | `/analysis/impact` 仅允许能力复用，不作为本 phase completion bar，避免扩大 server redesign blast radius |
</threat_model>
