# 08-02 Plan

## Goal

加固插件 runtime 的失败隔离和真实 CLI 可用性，确保插件问题被结构化暴露，而不是让主流程静默失效或直接崩溃。

## Tasks

1. 在 `PluginRegistry` 中把 initialize / analyze / generate 失败统一收口为 `PluginDiagnostic`。
2. 修复 `PluginLoader` 的 built-in explicit load / directory scan 重复加载问题，并支持按名字显式加载内置插件。
3. 确保 `pluginDir` 经 config loader 解析为绝对路径，修复真实 CLI user plugin 加载失败。
4. 给 plugin-generated files 增加路径越界保护，并把写入失败也写成 diagnostics。
5. 扩展回归测试，覆盖 failing user plugin 隔离与 CLI 端到端 fixture。

## DoD

- `PLG-06` 满足
- 插件失败会以结构化 diagnostics surfaced
- built-in 与 user plugin 都能通过真实 CLI fixture 证明可用
