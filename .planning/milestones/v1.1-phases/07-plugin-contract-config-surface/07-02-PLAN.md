# 07-02 Plan

## Goal

把新的配置契约真正接到 CLI 入口上：`generate` 不再忽略配置文件，并能在进入分析前拒绝非法插件配置，为 Phase 08 的运行时接入准备清晰 seam。

## Tasks

1. 更新 `src/cli/commands/generate.ts`，让其在 CLI 显式参数之外读取 normalized config，并优先做插件配置校验。
2. 锁定 precedence 与 lifecycle seam：CLI flags 优先于 config defaults；插件配置在本阶段只做校验/透传，不真正执行插件。
3. 扩展 `src/cli/commands/__tests__/generate.test.ts` 与 `src/cli/__tests__/generate.test.ts`，覆盖 config file 输入、legacy config fallback、错误消息与 precedence 行为。
4. 给 docs guardrail 补最小插件配置 drift 检查，确保配置文件名、配置段和 canonical example 不会再次背离实现。
5. 运行 CLI spot-check、`docs:check`、`typecheck/build`，确认 Phase 07 可无缝交接给 Phase 08。

## DoD

- `PLG-03` 满足
- `generate` 入口不再忽略配置文件中的插件配置
- 非法插件配置在 CLI / config loading 阶段给出清晰错误
- 回归测试、`npm run docs:check`、`npm run typecheck`、`npm run build` 通过
