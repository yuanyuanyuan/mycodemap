# 07-01 Plan

## Goal

定义插件配置 contract、默认值与 schema，使 `mycodemap.config.json` 成为真实且可验证的插件配置入口，而不是孤立类型或未来占位。

## Tasks

1. 扩展 `src/interface/config/index.ts`，为 `CodemapConfig` 增加正式插件配置类型，并尽量复用现有 `PluginLoadOptions` 语义。
2. 更新 `mycodemap.config.schema.json` 与 `src/cli/commands/init.ts` 默认配置，让 schema、默认值、配置文件名主叙事保持一致。
3. 新增 CLI 侧 config loader / normalizer，复用 `resolveConfigPath()` 读取新旧配置文件，合并默认值并对插件字段做显式校验。
4. 为配置路径回退、字段归一化、非法插件配置 hard-fail 增加 focused tests（至少新增 `src/cli/__tests__/config-loader.test.ts`）。
5. 同步最小配置文档入口，避免新增插件字段后 canonical config example 继续漂移。

## DoD

- `PLG-01`、`PLG-02` 满足
- `CodemapConfig` / schema / init default config / config loader 成为单一事实源
- focused tests、`npm run docs:check`、`npm run typecheck` 通过
