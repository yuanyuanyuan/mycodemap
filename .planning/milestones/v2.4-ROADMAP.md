# Milestone v2.4: parser-multilang-depth

**Status:** ✅ SHIPPED 2026-05-10
**Phases:** 67-69, 71
**Total Plans:** 8 completed

## Overview

在 `v2.2` 已收敛 parser registry 路由和 `v2.3` 已建立 graph-native schema 的基础上，`v2.4` 把 Python 解析从 regex-based MVP 升级为 Tree-sitter AST-based 深度解析，并建立多语言 parser 切换机制，让 CodeMap 对 Python 项目的分析质量开始接近 TypeScript 基线。

## Milestone Summary

**Key Decisions:**

- Chose WASM-first Python grammar loading because native `tree-sitter-python` was ABI-incompatible in this environment.
- Locked `PY-04` to explicit failure when Python grammar is unavailable instead of silent regex fallback.
- Reused the shared top-level `typeInfo` contract for Python enhancement output instead of inventing a Python-only side channel.
- Deferred `Phase 70` / `PY-07` / `PY-08` to `v2.5+` rather than stretching `v2.4` past the closed parser/type seams.

**Issues Resolved:**

- Replaced regex-based Python main-path parsing with AST-first parsing.
- Unified TS/JS/Python Tree-sitter switching behind one shared parser capability.
- Proved Python type enrichment survives compatibility and analyzer conversion into graph-facing output.
- Removed active parser legacy adapter shims and Core→Infrastructure construction coupling.

**Issues Deferred:**

- Python call-graph and Python complexity metrics (`PY-07`, `PY-08`)
- Hub/bridge detection, hook mechanism, node dedup
- Parser extension for Rust/Java/C++ and broader architecture-intelligence work

**Technical Debt Incurred:**

- `67` and `68` still do not have standalone `SECURITY.md` artifacts, though no open security blocker was found in milestone audit.
- Git range was not captured as a clean isolated slice because `v2.4` work finished on a dirty workspace with mixed in-flight changes.
