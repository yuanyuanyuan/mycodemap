# Milestone v2.2: architecture-foundation

**Status:** SHIPPED 2026-05-07
**Phases:** 59-62
**Total Plans:** 12

## Overview

把 CodeMap 的 parser、storage 和 MCP 执行表面收敛为更少、更稳定的架构基线。先消除 `fast` / `hybrid` parser 双轨、`filesystem` / `kuzudb` storage 漂移，以及 contract-backed MCP tools 的 `cli_redirect` 假执行，再在稳定执行语义之上补 `codemap_context` routing gate。

## Phases

### Phase 59: Parser Cutover

**Goal:** Collapse parser selection to a single Tree-sitter-based main path and remove threshold-driven `fast` / `hybrid` runtime behavior from the active analysis flow.
**Depends on:** None
**Plans:** 4 plans including 1 gap-closure follow-up

Plans:

- [x] 59-01: Parser orchestrator truth: `createParser()`, analyzer entry, mode deprecation contract
- [x] 59-02: Registry integration for TypeScript/JavaScript, Python, and Go main-path routing
- [x] 59-03: WASM fallback verification, deprecated-mode failure coverage, docs/runtime truth sync
- [x] 59-04: Server/API deprecated parser mode rejection aligned to the CLI-side actionable error contract

**Details:**
- Analyzer and legacy `createParser()` now share one registry-backed parser truth.
- Deprecated parser modes fail with a stable `DEPRECATED_PARSER_MODE` contract across CLI and server surfaces.
- TS/JS, Python, and Go now enter the same parser orchestration flow.
- WASM fallback and public docs now describe the shipped single-path parser story.

### Phase 60: Storage Convergence

**Goal:** Make SQLite the only supported persistent storage family and turn cross-backend fallback into explicit, actionable failure semantics.
**Depends on:** Phase 59
**Plans:** 3 plans in 2 waves

Plans:

- [x] 60-01: StorageFactory truth: remove `filesystem` / `kuzudb` active support, redefine `auto`
- [x] 60-02: Config/default/runtime migration: config-loader, defaults, error codes, remediation text
- [x] 60-03: SQLite-family fallback validation across native, `sql.js`, and `node:sqlite` paths

**Details:**
- Storage runtime truth now resolves only to SQLite-family persistence or explicit `memory`.
- Deprecated backend requests fail with deterministic `UNSUPPORTED_STORAGE_TYPE` remediation.
- Native sqlite failure remains observable and recoverable only inside the SQLite family.
- Config defaults, doctor guidance, and public docs all align to SQLite-only persistence.

### Phase 61: MCP Direct Execution

**Goal:** Replace `cli_redirect` contract-tool responses with real execution for an initial high-value MCP tool set, using shared command/service seams instead of duplicated logic.
**Depends on:** Phase 60
**Plans:** 3 plans in 3 waves

Plans:

- [x] 61-01: Shared execution seam for the first high-value contract-backed tools
- [x] 61-02: MCP adapter conversion: structured success/error output replaces `cli_redirect`
- [x] 61-03: Thin CLI wrapper cleanup, integration tests, and failure-path evidence

**Details:**
- `query` / `deps` / `analyze` now share `src/execution/contract-tools/` execution truth.
- `codemap_query`, `codemap_deps`, and `codemap_analyze` execute in-process and return structured envelopes.
- CLI wrappers remain thin and no longer own the family’s business execution logic.
- Architecture/output docs now describe the shipped shared executor + direct MCP execution boundary.

### Phase 62: Context Routing Gate

**Goal:** Add a minimal `codemap_context` routing gate on top of the stabilized execution surface so agents can request lightweight review/debug/default context and next-step tool suggestions.
**Depends on:** Phase 61
**Plans:** 2 plans in 2 waves

Plans:

- [x] 62-01: `codemap_context` contract, graph stats/risk summary, task-based suggestions
- [x] 62-02: `detail_level`, tool filtering, focused verification, docs sync

**Details:**
- `codemap_context` ships as a native MCP tool for `review` / `debug` / `default`.
- The routing payload includes graph stats, risk summary, and executable `nextToolSuggestions`.
- `minimal` / `standard` detail levels and strict `allowedTools` filtering are part of the contract.
- Docs and focused tests prove bounded routing without leaking unrelated tools.
