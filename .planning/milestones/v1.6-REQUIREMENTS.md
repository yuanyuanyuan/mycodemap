# Requirements: v1.6 CodeMap CLI dogfood reliability hardening

**Defined:** 2026-04-18 (frozen on 2026-04-19 from `ROADMAP.md` + Phase 25 summaries / verification / validation evidence)
**Core Value:** 为人类与 AI / Agent 提供可信的代码上下文、设计交接边界与后续演化决策依据。
**Previous Completed Milestone:** `v1.5` historical branch was closed; `Phase 25` starts the new `v1.6` line.

## v1.6 Requirements

### Analyze Failure Truth

- [x] **P25-SC1**: `analyze -i find` 在 scanner failure 时不能再伪装成可信的空成功结果
- [x] **P25-SC2**: `analyze find` 的 include/exclude 与 fallback discovery 必须对齐 config-aware scanning truth，而不是使用漂移的内部 glob
- [x] **P25-SC3**: stdout-only JSON 消费方必须能区分真实 0 命中与 `partialFailure` / failure

### Adjacent Dogfood Machine Contracts

- [x] **P25-DOGFOOD-COMPLEXITY**: `complexity -f <file> --json` 必须只返回目标文件，而不是全项目 `modules` 列表
- [x] **P25-DOGFOOD-CI-RISK**: `ci assess-risk --json` 必须提供 `passed` / `failed` / `skipped` 状态 truth
- [x] **P25-DOGFOOD-WORKFLOW**: `workflow start --json` 必须输出最小可解析 machine surface，同时不扩写 workflow 产品边界

### Docs Truth and Guardrails

- [x] **P25-SC4**: AI-facing command guidance 必须反映 repaired `analyze find` diagnostics 与相邻 dogfood JSON contract
- [x] **P25-SC5**: output schema 与 docs guardrail 必须锁住 success / `partialFailure` / failure 语义，并保持 `query -S` 作为稳定默认查找路径
- [x] **P25-DOCS**: `AI_GUIDE.md`、`docs/ai-guide/COMMANDS.md`、`docs/ai-guide/OUTPUT.md`、相关规则文档与校验脚本必须与实现同步

## Deferred

- `analyze -i find` degrade path 的 `stderr` 噪声治理
- 全 CLI surface 的统一旗标 / 统一 schema；`v1.6` 仅修复 dogfood-blocking 缺口

## Out of Scope

| Feature | Reason |
|---------|--------|
| 恢复 `Phase 22-24` 或重新引入 Docker / ArcadeDB | 与 `v1.6` 的 CLI dogfood reliability 版本线无关 |
| 借 `Phase 25` 一次性重做所有 CLI 命令的统一输出体系 | 本 milestone 只收口最阻塞 Agent 的契约缺口 |
| 把 `rtk` 变成 CodeMap 产品功能 | `rtk` 只属于执行包装层，不属于产品面 |
