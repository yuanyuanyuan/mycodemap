<objective>
Complete Phase 39 by turning publish-status polling into a standalone, read-only follow-up contract that humans and agents can use after `/release` without thickening the release authority surface.

Purpose: `Phase 38` already proved a Codex-side release adapter while preserving single-authority release governance. `RELF-02` is still open. This plan closes that gap with the smallest brownfield-safe addition: a one-shot publish-status command that reuses existing workflow-run truth, emits both terminal and structured output, and fails conservatively when exact run truth is unavailable.
Output: A standalone `publish-status` CLI entry, strict truth-first snapshot reporting, release/docs routing that frames it as follow-up observability only, passing focused validation, and planning truth that moves the milestone to Phase 40.
</objective>
