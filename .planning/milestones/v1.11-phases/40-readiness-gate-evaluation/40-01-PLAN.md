<objective>
Complete Phase 40 by refactoring the release quality check system from a two-layer mustPass/shouldPass model into a unified three-layer gate model (hard / warn-only / fallback), and exposing it as a standalone `readiness-gate` CLI command that `/release` can invoke.

Purpose: `Phase 38` and `39` built release entry surfaces and follow-up observability. `RELF-03` is still open. This plan closes that gap by making the readiness boundary explicit and verifiable: hard gates block, warn-only gates degrade confidence, and fallback gates halt automation when signals are unavailable.
Output: Refactored gate-rule engine, standalone CLI command with dual output modes, updated ship pipeline, release docs routing, passing validation, and planning truth that completes the milestone.
</objective>
