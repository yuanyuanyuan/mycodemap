# Milestone v2.6: polish-and-stabilize

**Status:** ✅ SHIPPED 2026-05-11
**Phases:** 79-83
**Total Plans:** 5

## Overview

在不重开新能力主线的前提下，`v2.6` 把 CodeMap 的一组“打磨收尾”问题收口为已验证的 shipped baseline：统一 complexity truth、清理 MCP stdio 空白行噪音、稳定 edge ID、补齐 Interface Contract `1.0.0`，并为高频 graph read path 加上 bounded performance optimization。

## Phases

### Phase 79: Complexity Truth Unification

**Goal**: 用户在 TS/JS/Python/Go 主路径与 downstream surfaces 中看到的 complexity metrics 都来自统一 canonical analyzer，而不是多套并行实现。  
**Depends on**: Shipped parser/type/deep-analysis baselines (`v2.4` + `v2.5`)  
**Plans**: 1 plan

Plans:

- [x] 79-01: unify active complexity truth on one canonical analyzer seam

**Details:**
- Complexity calculation now has one canonical implementation path for active TS/JS/Python/Go parser flows.
- Default CLI complexity reads canonical truth and fails explicitly when that truth is unavailable.
- Deprecated smart-parser complexity behavior is locked to the canonical analyzer to prevent silent drift.

### Phase 80: MCP Stdio Blank-Line Filtering

**Goal**: MCP stdio transport 会在输入边界过滤空白行，避免 blank-line noise 造成伪请求、协议报错或 transport-level drift。  
**Depends on**: Shipped MCP direct-execution baseline (`v2.2`)  
**Plans**: 1 plan

Plans:

- [x] 80-01: add repo-owned stdio transport seam and filter blank-line noise

**Details:**
- Blank input lines are ignored before request parsing on stdio transport paths.
- Malformed non-blank payloads now surface as explicit JSON-RPC parse-error frames.
- Valid MCP requests continue to work through the same startup path without contract regressions.

### Phase 81: Edge ID Normalization

**Goal**: Graph edge IDs across generation, storage, query, and impact paths use one stable normalized form so repeated runs and consumer paths stop seeing semantically identical but textually drifting IDs.  
**Depends on**: Shipped graph schema / impact baseline (`v2.3`)  
**Plans**: 1 plan

Plans:

- [x] 81-01: canonicalize graph edge IDs across generation, SQLite truth, and consumer paths

**Details:**
- Edge IDs now normalize with one lowercase / underscore-safe canonical rule.
- Repeated generate/save/load/query paths expose stable IDs for the same graph semantics.
- Legacy persisted IDs are backfilled at initialize time without reopening schema scope or losing graph truth.

### Phase 82: Interface Contract 1.0.0

**Goal**: 核心命令的 interface contract 补全 `outputShape`、`errorCodes`、`examples` 并声明 `stable: true`，形成可发布的 `1.0.0` 稳定表面。  
**Depends on**: Shipped schema-driven CLI contract baseline (`v2.0`) plus later command families  
**Plans**: 1 plan

Plans:

- [x] 82-01: promote the interface contract to `1.0.0` and require explicit stability declarations

**Details:**
- In-scope core commands now declare complete output/error/example fields plus `stable: true`.
- Full interface contract version is `1.0.0` across registry, `--schema`, and MCP schema adaptation.
- Missing contract fields are now caught by stricter schema/test gates before ship.

### Phase 83: Query Path Performance Optimization

**Goal**: 高频多跳 graph read path 在保持 SQLite-only truth 与现有结果语义不变的前提下，通过 bounded cache 和内存邻接表降低重复查询延迟。  
**Depends on**: Shipped graph schema / impact / community baselines (`v2.3`)  
**Plans**: 1 plan

Plans:

- [x] 83-01: add bounded eager-cache read acceleration without changing graph truth semantics

**Details:**
- Repeated query/impact-style read paths reuse prebuilt read indexes and bounded in-memory impact cache.
- QueryHandler no longer performs a second BFS after storage returns layered impact truth.
- Degraded or cache-miss paths still fall back safely to SQLite truth instead of serving stale data.

## Milestone Summary

**Key Decisions:**

- Complexity truth was centralized into one canonical analyzer seam instead of preserving per-parser behavior.
- Blank-line filtering was placed at the stdio transport boundary, not inside MCP business logic.
- Edge identity was derived from stable semantic references instead of random module/symbol IDs.
- Contract stability became explicit schema truth via required `stable: true`.
- Performance work stayed retrieval-safe: eager acceleration is allowed, but SQLite remains the fallback truth.

**Issues Resolved:**

- Removed active complexity output drift between parser paths and legacy surfaces.
- Eliminated blank-line protocol noise on MCP stdio startup.
- Stopped semantically identical graph edges from drifting textually across generate/save/load/query paths.
- Promoted the current core contract surface to a test-gated `1.0.0`.
- Removed redundant graph-walk work from the high-frequency impact read path.

**Issues Deferred:**

- Pre-existing `codemap_env_contract` native-vs-contract rename/skip warnings remain visible in MCP registration tests.
- Broader contract migration and output-surface expansion remain future work (`AGENT-10`, `AGENT-11`).
- Architecture-intelligence and broader parser coverage stay deferred to `v3.0+`.

**Technical Debt Incurred:**

- Verification and validation artifacts for v2.6 were backfilled during closeout rather than generated immediately after each execute-phase run.
