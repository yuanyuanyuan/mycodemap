# Requirements: CodeMap v2.3 graph-capability

**Defined:** 2026-05-07
**Core Value:** 为人类与 AI / Agent 提供可信的代码上下文、设计交接边界与后续演化决策依据。

## v2.3 Requirements

### Graph Schema

- [x] **GRAPH-01**: 维护者运行 graph generation 时，系统会把 symbol、file、dependency 与 analysis metadata 持久化到 graph-optimized SQLite schema，而不是继续依赖为 governance 场景设计的旧结构
- [x] **GRAPH-02**: 现有 `generate` / `query` / `deps` / `analyze` 成功路径可以读取新的 graph schema，而不改变既有 CLI / MCP 成功输出契约
- [x] **GRAPH-03**: persisted edges 包含 `EXTRACTED`、`INFERRED`、`AMBIGUOUS` confidence 语义，供后续 impact / clustering / agent surfaces 读取

### Incremental Graph Updates

- [x] **INCR-01**: 维护者可以基于 `git diff` 或显式 changed-file set 触发 scoped incremental update，而不是每次都全量重建整个 repo graph
- [x] **INCR-02**: incremental update 会明确返回或记录哪些节点/边被重算、复用或失效，便于诊断缓存/传播错误

### Impact Analysis

- [x] **IMPT-01**: 维护者可以从 file 或 symbol 入口执行 recursive impact traversal，并获得分层的 downstream reachability 结果
- [x] **IMPT-02**: impact analysis 输出会区分 direct impact 与 transitive impact，并提供 layered summary 便于人类或 agent 快速消费

### Community Detection

- [x] **COMM-01**: 维护者可以基于已存储 graph 计算基础 community / cluster 结果，用于揭示模块边界而不只看 import chain
- [x] **COMM-02**: community detection 结果通过至少一个现有 CLI 或 agent-facing surface 暴露，而不要求调用方直接读取原始 SQL 表

## v2.4+ Requirements

### Agent Graph Experience

- **AGX-01**: surprise score (`report` mode)
- **AGX-02**: execution flow trace (two-phase)
- **AGX-03**: bare-name resolution

### Deep Analysis Hooks

- **HOOK-01**: hub / bridge detection
- **HOOK-02**: hook mechanism (first-remind-then-silent, Phase 58 integration)
- **HOOK-03**: node dedup (3-layer)

### Architecture Intelligence / Agent Integration

- **INT-04**: Auto-Provisioned Agent Skills
- **INT-05**: MCP `verify_contract` Tool
- **ARCH-01**: Auto-Generate design.md from codebase
- **ARCH-02**: Auto-Generate Architecture Remediation Patches
- **ARCH-03**: Self-Healing Design Contract (Drift Approval)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Reopening parser main-path convergence | `v2.2` already closed parser baseline; `v2.3` should build on it, not relitigate it |
| Reopening SQLite-only / MCP direct-execution convergence | Those belong to `v2.2` closeout unless a concrete graph blocker forces a focused follow-up |
| Surprise score / execution flow trace / bare-name resolution | These are explicitly deferred to `v2.4+` agent graph experience |
| Hook mechanism / hub-bridge detection / node dedup | These are explicitly deferred to `v2.5` deep-analysis-hooks |
| Auto-Provisioned Agent Skills / `verify_contract` | These remain `v3.0+` agent integration work, not graph-core capability |
