# Milestone v2.7: agent-effectiveness-validation

**Status:** ✅ SHIPPED 2026-05-11
**Phases:** 75-78
**Total Plans:** 4 completed

## Overview

`v2.7` 为 CodeMap 补齐了第一套 agent-effectiveness validation 基础设施：新建 `codemap agent-metrics` 命令家族，围绕固定代表性查询样本输出 token 成本、报告聚合、CI threshold gate，以及 latest-vs-previous 趋势 / 分布 / 高成本 advisory。整个 milestone 维持 CLI-first、SQLite truth、shared output、advisory-only intelligence 的收敛方向，不引入第二套 measurement 或 gate surface。

## Phases

### Phase 75: Core Infrastructure and Basic Token Analysis ✓ COMPLETE (2026-05-10)

**Goal:** 用户可以通过 `codemap agent-metrics token` 对代表性 CodeMap 查询执行 token 成本分析，获得每个查询的响应大小、估算 token 数和原始字符数。  
**Depends on:** Phase 74 stable delegated-work baseline  
**Requirements:** `CMD-01`, `CMD-02`, `TOKEN-01`, `TOKEN-02`  
**Plans:** 1 — complete

**Success Criteria:** ALL MET
1. `codemap agent-metrics token` executes representative CodeMap queries and reports per-query token-cost data. ✓
2. Each query result includes response JSON size, estimated token count, and raw character count. ✓
3. No-arg `codemap agent-metrics` closes to the report flow. ✓
4. The command family stays thin over a service layer with SQLite persistence. ✓

### Phase 76: Estimation and Reporting ✓ COMPLETE (2026-05-10)

**Goal:** 用户可以获得格式化的 token 成本报告，支持人类可读表格和机器可消费的 JSON 两种输出模式，并按查询类型查看分组统计。  
**Depends on:** Phase 75  
**Requirements:** `RPT-01`, `RPT-02`, `RPT-03`  
**Plans:** 1 — complete

**Success Criteria:** ALL MET
1. `codemap agent-metrics report` outputs a human-readable formatted report with summary and tables. ✓
2. `--json` output exposes a stable machine-readable contract including `schemaVersion` and `rawCharCount`. ✓
3. Grouped query-type summary truth is available in the report surface. ✓

### Phase 77: CI Gate and Threshold Enforcement ✓ COMPLETE (2026-05-10)

**Goal:** CI 管道可以通过 `--max-tokens-per-query` 阈值参数检测 token 成本过高的查询，默认以 warn-only 模式运行，不阻断 pipeline。  
**Depends on:** Phase 76  
**Requirements:** `CI-01`, `CI-02`, `CI-03`  
**Plans:** 1 — complete

**Success Criteria:** ALL MET
1. Explicit threshold failures return non-zero exit code on report/root paths. ✓
2. CI-oriented output exposes concise pass/fail context with key metrics. ✓
3. Default no-threshold behavior remains visible warn-only rather than blocking. ✓

### Phase 78: Intelligence Layer — Trends and Distribution ✓ COMPLETE (2026-05-11)

**Goal:** 用户可以追踪 token 成本的绝对趋势、识别成本最高的查询类型、以及查看按查询类型的分布统计，从而判断哪些场景的 token 消耗可能抵消调用次数减少的收益。  
**Depends on:** Phase 75 stable token baseline, Phase 77 gate contract  
**Requirements:** `TOKEN-03`, `TOKEN-04`, `TOKEN-05`  
**Plans:** 1 — complete

**Success Criteria:** ALL MET
1. Report tracks latest-vs-previous absolute token trends by query type. ✓

3. Report provides grouped `p50/p95/max` distribution depth for outlier detection. ✓

## Milestone Summary

**Key Decisions:**

- `agent-metrics` 作为独立命令家族落地，而不是扩展 `benchmark`。
- report 聚合 truth 保持在 service layer，human/JSON 两种输出共享一条 read-time aggregation 路径。
- CI gate 只在 CLI 边界基于显式阈值决定非零退出；默认行为保持 warn-only。
- Phase 78 intelligence 只做 additive advisory，不改变 Phase 77 gate verdict 或 exit-code 语义。

**Issues Resolved:**

- CodeMap 首次具备针对代表性查询集的 token-cost measurement truth，而不再只依赖主观判断。
- `agent-metrics report` 具备 grouped summary、latest-run-only semantics 和清晰的人类可读输出。
- 单条查询超阈值的 CI 风险可见且可阻断，但不会强行引入未经校准的默认 hard gate。
- runtime `dist/` 在 Phase 78 closeout 前被重建，built CLI 输出重新与源码 truth 对齐。

**Issues Deferred:**

- `POL-01` ~ `POL-04` 继续保留在 `v2.6 polish-and-stabilize`
- `INTEL-01` ~ `INTEL-04` 保留为更后续的 intelligence 深化项
- `AGENT-10` / `AGENT-11` 仍属于后续 contract migration / shared output 扩展

**Technical Debt Incurred:**

- token estimation 仍是 heuristic，而非 model-specific tokenizer exact truth；需要后续真实数据校准。
- 默认 gate 阈值尚未固定，仍需 baseline run 支持校准决策。
- closeout 发生在 dirty workspace 上，未捕获干净隔离的 git range / diff stats。
- 仓库级 `rtk npm run docs:check` 仍有与 `agent-metrics` 无关的既有 AI-doc guardrail 失败。
