<threat_model>
Threats:
- Agent passes a file path to `complexity` but receives all modules, causing context explosion.
- Agent cannot tell whether `ci assess-risk` passed, failed, or skipped without parsing prose.
- `workflow start --json` accidentally implies workflow product expansion beyond existing analysis-only phases.
Mitigations:
- Keep `complexity -f` output to a single `file` object.
- Add explicit JSON status fields to `ci assess-risk`.
- Keep `workflow start --json` additive and limited to existing context id/currentPhase/template/nextSteps.
Residual risk:
- Human output for these commands remains mostly existing prose; Plan 25-03 documents machine-first invocation patterns.
</threat_model>
