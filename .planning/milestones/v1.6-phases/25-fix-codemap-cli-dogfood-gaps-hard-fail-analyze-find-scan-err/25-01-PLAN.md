</objective>

<threat_model>
Threats:
- A machine consumer trusts `results=[]` while scanner failed, causing false-negative code analysis.
- A fallback search widens beyond `mycodemap.config.json` include/exclude and introduces noisy false positives.
- stderr-only diagnostics disappear in agent integrations that parse stdout JSON only.
Mitigations:
- Make `diagnostics.status` part of stdout `CodemapOutput`.
- Keep fallback discovery inside `loadCodemapConfig()` + `discoverProjectFiles()` boundaries.
- Add tests for `partialFailure`, `failure`, and true zero-hit success.
Residual risk:
- Existing external consumers that assume `CodemapOutput` has no `diagnostics` field should tolerate additive optional JSON fields; docs update is handled in Plan 25-03.
</threat_model>
