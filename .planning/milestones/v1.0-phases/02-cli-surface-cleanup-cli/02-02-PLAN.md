<objective>
为四个 removed commands 提供清晰、最小 blast-radius 的兼容/错误策略，满足“迁移平滑”而不是“静默消失”。

Purpose: 如果只把命令从 help 中移除，旧调用会直接退化成 generic unknown command；这不符合设计稿的迁移原则，也不利于真实使用者定位问题。
Output: 增加 removed-command 提示入口与测试，确保 `server`、`watch`、`report`、`logs` 的调用都有明确反馈。
</objective>
