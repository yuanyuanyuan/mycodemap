## Overview

`v2.7.1` 把 CodeMap 的 installable `git commit` hooks 收口成更适合 AI Agent 的控制协议层。它先在 `Phase 84` 建立 fail-fast structured blocker protocol，再在 `Phase 85` 继续降低 mixed output 噪音、补齐 protocol-only mode、显式 log-path fallback 和更清晰的 status taxonomy。

## Milestone Summary

**Key Decisions:**

- Hook truth remains in `scripts/hooks/templates/*`; managed `.mycodemap/hooks/*` copies are parity targets.
- Human-readable diagnostics remain useful by default, but `CODEMAP_PROTOCOL_ONLY=1` is the preferred mode for fully autonomous agents.
- Status semantics matter to agents: `not_applicable` is distinct from `skipped`, and report-only limit-reached is a `warn`, not a failure.

**Issues Resolved:**

- AI agents no longer need to re-translate every hook failure from natural language into actions.
- Commit-message failures now provide the same machine-readable contract quality as pre-commit failures.
- Mixed human/protocol output now has an explicit low-noise escape hatch and log-path recovery story.
- Shared hook protocol source now lives in `scripts/hooks/templates/protocol-contract.json`, so tests and AI docs no longer need to hardcode schema/env tokens independently.

**Issues Deferred:**

- Broader architecture-intelligence work remains future scope (`v3.0+`).
- Contract migration and shared-output follow-ups (`AGENT-10`, `AGENT-11`) stay outside this milestone.

**Technical Debt Incurred:**

- Shell runtime still renders protocol JSON inline; the shared contract source now exists, but future work may still want generated shell fragments if more hook families appear.
