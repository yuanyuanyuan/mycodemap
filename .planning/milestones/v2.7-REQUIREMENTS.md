# Requirements: CodeMap v2.7 agent-effectiveness-validation

**Defined:** 2026-05-10
**Closed:** 2026-05-11
**Core Value:** 为人类与 AI / Agent 提供可信的代码上下文、设计交接边界与后续演化决策依据。

## Shipped Requirements

### Token Analysis

- [x] **TOKEN-01**: 命令能对一组代表性 CodeMap 查询执行 token 成本分析
- [x] **TOKEN-02**: 每个查询报告响应 JSON 大小、估算 token 数与原始字符数
- [x] **TOKEN-03**: 按查询类型追踪绝对 token 成本趋势
- [x] **TOKEN-04**: 识别 token 成本最高的查询类型并提供 advisory
- [x] **TOKEN-05**: 按查询类型提供 `p50/p95/max` 分布统计

### Report & Output

- [x] **RPT-01**: 提供人类可读的格式化报告（表格 + 摘要）
- [x] **RPT-02**: 提供 JSON 输出模式（`--json`），包含 `schemaVersion` 和 `rawCharCount`
- [x] **RPT-03**: 报告包含按查询类型分组的汇总统计

### CI Integration

- [x] **CI-01**: 支持 `--max-tokens-per-query` 阈值参数，超阈值返回非零退出码
- [x] **CI-02**: CI 模式下输出简洁的 pass/fail 摘要与关键指标
- [x] **CI-03**: 默认保持 warn-only，不直接阻断 pipeline

### Command Structure

- [x] **CMD-01**: 命令入口为 `codemap agent-metrics`，子命令为 `token` / `report`
- [x] **CMD-02**: 无参数运行时等价于完整 report flow

## Out of Scope

| Feature | Reason |
|---------|--------|
| A/B harness（CodeMap-enabled vs text-search-only） | 需要额外定义 ground truth，超出本 milestone |
| Agent 行为分类（accepted/re-queried/abandoned） | 需要 MCP gateway 持续数据，不属于 CLI-only MVP |
| Adoption Decay 持续监控 | 需要持续采集，不属于离线 CLI 报告 |
| Precision-Weighted Cost Model | 缺少真实 agent session 数据校准 |
| 跨工具成本对比（rg/grep） | 信息密度不同，直接对比会误导 |
| MCP gateway 持续采集 | 明确递延到后续 milestone |
