# Milestone v2.0: agent-native-foundation

**Status:** SHIPPED 2026-05-01
**Phases:** 40.1-49
**Total Plans:** 16

## Overview

把 CodeMap 从"人类 CLI + AI 可用"升级为"AI-Native 优先、人类友好的代码架构治理基础设施"。用机器可读契约统一 CLI / MCP / 文档三层表面，用结构化错误和持续诊断修复信任危机，用 WASM 回退消除安装失败的头号 drop-off。

### Phase 45: Failure-to-Action Protocol

**Goal:** Turn every error into a structured, recoverable state transition.
**Depends on:** Phase 44
**Plans:** 3 plans

Plans:

- [x] 45-01: ActionableError type system and error codes
- [x] 45-02: Auto-remediation engine and WASM fallback handler
- [x] 45-03: Command migration to ActionableError

**Details:**
- Errors return `{attempted, rootCause, remediationPlan, confidence, nextCommand}`
- Native dependency failures auto-suggest `--wasm-fallback` or prebuilt binary URL
- Agents can attempt remediation without human intervention (`--apply-suggestion` flag)
- Confidence scoring (>= 0.8 threshold) prevents cascading failures
- No auto-execution without explicit `--apply-suggestion`
- 33 tests across apply-suggestion, wasm-fallback, and errors test files

## Milestone Summary

**Key Decisions:**

- v2.0 scope filtered to 10 phases from 19 raw ideas — selected by highest confidence × leverage
- Phase numbering continues from 40 (no reset)
- Auto-Generate design.md / Remediation / Self-Healing deferred to v3.0 "Architecture Intelligence"
- SQLite-only storage migration deferred — deserves standalone milestone
- First-Run Concierge / Zero-Config Preview deferred to v2.1
- Progressive migration for contract schema — start with 3 commands, expand over time

**Issues Resolved:**

- Ghost commands (`check:architecture`, `check:unused` echo stubs) removed
- Native dependency install failures now have WASM fallback path
- CLI output inconsistency (bolt-on `--json`) replaced with unified output infrastructure
- MCP tool registration no longer requires handwritten maintenance per command
- Errors are now structured state transitions instead of dead ends

**Issues Deferred:**

- First-Run Concierge + Bootstrap Profiles → v2.1
- Zero-Config Preview / Progressive Commitment → v2.1
- Auto-Provisioned Agent Skills → v2.2
- MCP `verify_contract` Tool → v2.2
- Auto-Generate design.md from codebase → v3.0
- Auto-Generate Architecture Remediation Patches → v3.0
- Self-Healing Design Contract → v3.0
- SQLite + In-Memory Graph Migration → v3.0
- Path-Scoped Governance → Continuous

**Technical Debt Incurred:**

- 12+ commands still need contract schema definitions (progressive migration)
- Global flags (`--wasm-fallback`, `--apply-suggestion`, `--native`) need full schema inclusion
- `benchmark` command uses direct `console.log` instead of shared output infrastructure
- `outputSchema` in MCP adapter describes `cli_redirect` shape, not contract's `outputShape`
- Zod default/optional ordering produces required properties for optional flags with defaults
