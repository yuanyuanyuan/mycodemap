## v1 Requirements

Requirements for `v1.10 governance-debt-cleanup`.

### Governance Drift Detection

- [x] **GOV-01**: 维护者运行现有 docs guardrail 时，可以自动发现 `AGENTS.md`、`CLAUDE.md`、`.claude/CLAUDE.md` 与 live docs 之间的重复政策 / competing authority drift
- [x] **GOV-02**: 维护者运行现有 docs guardrail 时，可以发现入口/导航文档引用了不存在、已废弃或不再推荐的命令/路径，并把它们标记为 ghost command / ghost route drift
- [x] **GOV-03**: 入口文档治理自审机制继续复用 `npm run docs:check` 与 `node dist/cli/index.js ci check-docs-sync`，而不是新增独立治理入口或平行流程

### Validation Trust

- [x] **TRUST-01**: 维护者可以从 `docs/rules/validation.md`、`docs/rules/engineering-with-codex-openai.md`、README 与 AI guide 读到同一套验证顺序与 docs guardrail 入口事实
- [x] **TRUST-02**: 维护者可以明确区分 `report-only` / `warn-only` / blocking 三类 gate 语义，不会把软 gate 写成 hard gate success
- [x] **TRUST-03**: milestone 交付至少包含一个 validation-trust 失败场景预演，证明相关 drift 会被 guardrail 或交叉检查显式发现

### Archive Identity

- [x] **ARCH-01**: 维护者可以从 `.planning/PROJECT.md`、`.planning/REQUIREMENTS.md`、`.planning/ROADMAP.md`、`.planning/STATE.md` 明确识别当前 active planning surface
- [x] **ARCH-02**: 维护者打开 archived milestone 文档时，可以明确识别它是 historical snapshot / readonly archive，而不是 current planning truth
- [x] **ARCH-03**: 维护者可以从 roadmap / state / retrospective / milestone index 一致判断某个 milestone 当前是 active 还是 archived，不会看到相互冲突的身份表达

## v2 Requirements

### Release Follow-ups

- **RELF-01**: 为非 Claude runtime 提供等价 release skill / command 包装
- **RELF-02**: 可选轮询 GitHub Actions 发布状态并输出结构化报告
- **RELF-03**: 在未来版本中评估是否把 release readiness 检查接入 CI 或 pre-release gate

### Extended Governance

- **GOVX-01**: 评估是否需要把 drift detection 从 entry docs 扩展到更广泛的 rules / AI docs inventory
- **GOVX-02**: 评估是否值得引入结构化文档 inventory / generation，以降低 snippet-based guardrail 的长期维护成本

## Out of Scope

| Feature | Reason |
|---------|--------|
| 本 milestone 内执行真实 `/release v1.9`、npm publish 或 GitHub Release | 发布仍是 L3 边界，且本 milestone 聚焦治理债而不是实际发布 |
| 为非 Claude runtime 立刻实现 release wrapper | 这是后续 release follow-up，不属于本次治理债主线 |
| 立刻接入 GitHub Actions 发布轮询或 CI readiness gate | 属于 release follow-up，当前先解决治理真相与 archive identity |
| 新增治理中间层、独立治理 CLI 或生成式文档系统 | 本 milestone 必须复用现有 docs guardrail / live docs，而不是再造入口 |
| 回填或重写所有历史 archive 文档 | 当前目标是修 active/archived 身份与导航边界，不是重做全部历史工件 |
