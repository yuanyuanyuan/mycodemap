# Requirements: CodeMap v2.4 parser-multilang-depth

**Defined:** 2026-05-09
**Closed:** 2026-05-10
**Core Value:** 为人类与 AI / Agent 提供可信的代码上下文、设计交接边界与后续演化决策依据。

## Shipped Requirements

### Tree-sitter Python Grammar

- [x] **PY-01**: `tree-sitter-python` WASM grammar 已安装并可在运行时加载，Python 文件的 AST 解析不再依赖 regex
- [x] **PY-02**: Python Tree-sitter parser 能正确提取 imports、exports、symbols、classes、functions、decorators、async 定义和嵌套结构

### Multi-language Parser Switching

- [x] **PY-03**: `TreeSitterParser` 支持按文件扩展名自动加载对应语言的 Tree-sitter grammar，不再硬编码 TypeScript
- [x] **PY-04**: parser registry 在 Python Tree-sitter grammar 可用时路由 `.py` 文件到 AST parser；grammar 不可用时显式失败并给出可操作错误，而不是静默回退到 regex parser

### Python Type Enhancement

- [x] **PY-05**: `PythonTypeEnhancer` 能从 Python docstring（Google/NumPy/Sphinx 风格）和 PEP 484 类型注解中提取类型信息
- [x] **PY-06**: 增强后的类型元数据与 `TypeScriptTypeEnhancer` 产出的形状一致，并通过共享顶层 `typeInfo` surface 进入当前 graph/module truth 输出

### Parser Architecture Cleanup

- [x] **PAR-09**: active runtime 统一使用 interface-layer `ParseResult` 契约，legacy `IParser` 仅保留兼容用途
- [x] **PAR-10**: `TreeSitterParser` 迁入 Infrastructure layer 并作为 TS/JS 主实现注册到 `ParserRegistry`
- [x] **PAR-11**: Core 通过 composition root 注入 parser registry / enhancer，不再直接构造 Infrastructure parser 依赖
