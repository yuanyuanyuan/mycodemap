# Milestone v1.1: 插件扩展点产品化

**Status:** ✅ SHIPPED 2026-03-24
**Phases:** 7-9
**Total Plans:** 6

## Overview

这个 milestone 不是发明新插件系统，而是把仓库里已经存在的 `src/plugins/` 骨架升级为正式产品面。  
执行顺序为：先收口配置契约与 CLI loader → 再把 runtime 接进 `generate` → 最后把 README / AI docs / guardrail / e2e 证据闭环。

## Milestone Summary

**Key Decisions:**
- 只有配置文件**显式声明** `plugins` 段时才启用插件 runtime，避免旧项目被默认启用 built-in plugin
- 用 `codemap.json.pluginReport` 与 `AI_MAP.md` 的 `Plugin Summary` 作为插件结果单一事实源
- docs guardrail 必须同时覆盖配置文件名、插件配置表、`pluginReport` 契约与 diagnostics 说明

**Issues Resolved:**
- 插件系统长期停留在 `src/plugins/` 内部骨架，缺少正式配置入口
- `generate` 过去忽略配置文件，插件配置无法进入主流程
- 插件失败之前主要依赖 logger，AI / CLI 无法稳定消费诊断结果
- 真实 CLI 下 user plugin 会因相对 `pluginDir` 路径而加载失败
- 文档与实现之间没有针对插件产品面的脚本级 guardrail

**Issues Deferred:**
- 插件 marketplace / 远程安装生态
- KuzuDB / Neo4j 后端生产化
- 是否需要独立 HTTP API 产品面

**Technical Debt Incurred:**
- 当前运行约束禁止 `git commit` / `git tag` / `git push`，因此 v1.1 只有 planning archive，没有正式 release tag
