# Milestone v2.1: ux-onboarding-enhancement

**Status:** ✅ SHIPPED 2026-05-06
**Phases:** 53-58
**Total Plans:** 15

## Overview

把 CodeMap 从“安装后需要手动配置”升级为“首次运行即能提供价值”的入门体验，让新用户零配置即可预览代码库，并逐步引导完成深度配置。Phase 58 是中途插入的验证依赖，因此先完成 Phase 58，再做 Phase 57，以避免把同一验证面重复做两次。

## Phases

### Phase 53: Bootstrap Profiles + Project Detection

**Goal:** Establish project type auto-detection and Bootstrap Profile system so first-time users get sensible defaults without manual configuration.
**Depends on:** None
**Requirements:** FRC-01, FRC-02, FRC-03, FRC-04
**Plans:** 3 plans in 2 waves

**Wave 1** *(no dependencies)*
- [x] 53-01-PLAN.md — Core detection + profile infrastructure (detect.ts, profile-loader.ts, 5 profile JSON files)
- [x] 53-02-PLAN.md — Profile plan integration (profile-plan.ts, reconciler.ts, init.ts --profile, interface contract)

**Wave 2** *(blocked on Wave 1 completion)*
- [x] 53-03-PLAN.md — Tests + first-run guide (detect.test.ts, profile-loader.test.ts, init-profile.test.ts, first-run-guide.ts update)

**Details:**
- `codemap` first run auto-detects project type (Node.js, Python, Go, Rust, generic)
- Each project type has a Bootstrap Profile with parser config, ignore patterns, and analysis depth defaults
- Interactive mode supports review/accept/skip of the recommended profile
- Profile definitions are stored as data files, not hardcoded in source
- Final verifier pass closed the `--profile` commander wiring gap and left only cosmetic cleanup debt behind

### Phase 54: Zero-Config Preview

**Goal:** Let users see CodeMap value immediately without writing any configuration file.
**Depends on:** Phase 53 (profile system provides fallback defaults)
**Requirements:** ZCP-01, ZCP-02, ZCP-03, ZCP-04
**Plans:** 2 plans in 2 waves

**Wave 1** *(no dependencies)*
- [x] 54-01-PLAN.md — Service modules: dependency extractor + complexity scanner + smol-toml install

**Wave 2** *(blocked on Wave 1 completion)*
- [x] 54-02-PLAN.md — Preview command + renderer + interface contract + CLI registration + tests

**Details:**
- `codemap preview` runs without `mycodemap.config.json` in empty or fresh projects
- Project type detection and Bootstrap Profile fallback provide initial config truth
- Output summarizes file count, module count, key dependencies, and complexity hotspots
- Preview ends with a `--save` hint rather than a discard flow

### Phase 55: Agent Bootstrap Assets

**Goal:** Generate assistant bootstrap assets during `mycodemap init` for both main agents (Claude/Codex parent sessions) and subagent retrieval infrastructure.
**Depends on:** None
**Requirements:** ABT-01, ABT-02, ABT-05, INI-01 (seeded, partial at close)
**Plans:** 2 plans in 1 wave

**Wave 1** *(no dependencies)*
- [x] 55-01-PLAN.md — Manifest extractors (package.json/pyproject.toml/go.mod/Cargo.toml) + env-contract seed generation
- [x] 55-02-PLAN.md — Assistant bootstrap asset generator (4 per-runtime files under .mycodemap/assistants/)

**Details:**
- `mycodemap init` creates `.mycodemap/env-contract.json` as the seed Project Environment Contract
- `mycodemap init` creates `.mycodemap/assistants/` with `claude-context.md`, `agents-context.md`, `claude-hook-example.json`, and `codex-agent-example.toml`
- `--profile claude|codex|generic` selects which platform adapter configs to generate
- `mycodemap init --json` remained a partial follow-up at close; receipt JSON wiring was not yet fully converged

### Phase 56: Init Receipt + Next Steps

**Goal:** Complete the init experience with clear receipt reporting that distinguishes main-agent setup from subagent setup, safe team-owned file handling, and synchronized documentation.
**Depends on:** Phase 55 (assets must exist before receipt can reference them)
**Requirements:** ABT-03, ABT-04, INI-02, INI-03
**Plans:** 3 plans

**Wave 1** *(no dependencies)*
- [x] 56-01-PLAN.md — Receipt enhancement: two-section layout, personalized next steps, sync detection

**Wave 2** *(blocked on Wave 1 completion)*
- [x] 56-02-PLAN.md — Tests: receipt classification, sync detection, personalized next steps coverage
- [x] 56-03-PLAN.md — Doc sync: README, SETUP_GUIDE, AI_ASSISTANT_SETUP unified flow

**Details:**
- Init receipt displays a Main Agent section and a Subagent section
- Team-owned files (`CLAUDE.md`, `AGENTS.md`) are never silently rewritten
- Next steps are personalized per asset, not a fixed three-step welcome template
- Setup docs now describe install → init → doctor → generate → connect agent

**Depends on:** Phase 58
**Requirements:** VER-01, VER-02, VER-03
**Plans:** 0 plans

### Phase 58: Subagent Environment Contract Retrieval

**Goal:** At the project layer (not the platform layer), automatically discover project-specific environment contracts scattered across hooks, entry docs, package scripts, and rule-control helpers, and make them retrievable by sub-agents via CLI or MCP.
**Depends on:** Phase 55, Phase 56, Phase 57
**Requirements:** ABT-01, ABT-02, ABT-03, VER-03, SDC-01, SDC-02, SDC-03, SDC-04, SDC-05
**Plans:** 5 waves

**Historical implementation waves**
- [x] 58-01 — Canonical env-contract schema, source discovery, agent filtering, and drift detection
- [x] 58-02 — CLI retrieval surface, MCP exposure, and JSON/human outputs
- [x] 58-03 — Assistant bootstrap guidance updated to retrieval-first wording
- [x] 58-04 — E2E coverage, init wiring, and evidence scaffolding
- [x] 58-05 — Verification re-baseline, manual Claude/Codex evidence capture, and closeout rollup

**Details:**

- The canonical contract is agent-type-sensitive and derived from executable truth
- Real Claude/Codex evidence was captured in `docs/generated/phase-58/subagent-evidence/`

## Milestone Summary

**Key Decisions:**
- Phase 58 was inserted mid-stream and completed first to avoid duplicate verification runs.
- Retrieval-first contracts replaced prompt-snippet injection for subagents.
- `mycodemap init` preserves user-owned config and records receipt/profile state without silent rewrites.

**Issues Resolved:**
- Bootstrap profile detection and idempotent reruns
- Two-section init receipt and personalized next steps
- Env-contract drift detection and manual subagent evidence rollup
- Built CLI verification across real temp project scenarios

**Issues Deferred:**
- `INI-01`: `mycodemap init --json` still lacked a fully machine-readable receipt output path at close
- `F-1`: legacy-config receipt wording remained a cosmetic follow-up if not already normalized elsewhere

**Technical Debt Incurred:**
- `init --json` contract / implementation mismatch remains a follow-up
- Future v2.2 work continues from the archived onboarding surface
