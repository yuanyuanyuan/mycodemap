# Milestone v1.0: AI-first 重构

**Status:** ✅ SHIPPED 2026-03-24
**Phases:** 1-6
**Total Plans:** 15

## Overview

这个 milestone 不是重写 CodeMap，而是把已经存在的 legacy CLI、分析管线、MVP3 架构和 guardrail 重新收束到一个 AI-first、机器可读、边界清晰的产品面。  
执行顺序为：定位基线 → CLI surface 收缩 → analyze 契约收口 → workflow 简化 → ci/ship 对齐 → 共享发现规则与文档护栏收口。

## Milestone Summary

**Key Decisions:**
- 采用 AI-first 代码地图定位，而不是继续容忍实现型 workflow / HTTP API / report surface 混入公共入口
- 将 `analyze` / `workflow` / `ci` / `ship` 的边界分别收口，并通过 docs guardrail 编码化
- 用共享 `.gitignore` 感知文件发现模块解决 analyzer / scanner 规则分叉

**Issues Resolved:**
- 入口文档、AI 文档、CLI surface 与实现长期漂移
- `analyze` 旧 8-intent public contract 难以被 AI 稳定消费
- workflow 把分析、实现、CI 混成单一产品面
- ship 与 ci 之间重复实现发布前检查
- `*.test.ts` 默认排除对嵌套文件失效

**Issues Deferred:**
- 扩展点 / 插件系统生产化
- KuzuDB / Neo4j 后端生产化
- 是否需要独立 HTTP API 产品面

**Technical Debt Incurred:**
- Phase 01 缺少 `VALIDATION.md`
- Phase 02 `VALIDATION.md` 仍为 draft
- 历史 archive/setup 文档尚未全部纳入当前 docs guardrail
