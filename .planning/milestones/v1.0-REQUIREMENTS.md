# Requirements Archive: v1.0 AI-first 重构

**Archived:** 2026-03-24  
**Outcome:** All v1 requirements validated

## v1 Requirements

### Product Positioning

- [x] `POS-01`: `README.md`、`AI_GUIDE.md` 与 AI 文档入口将 CodeMap 描述为 AI-first 代码地图工具
- [x] `POS-02`: 保留命令默认输出机器可读结果，并提供明确的人类可读模式说明
- [x] `POS-03`: 文档明确区分“Server Layer 架构概念”和公共 `server` CLI 命令

### Command Surface

- [x] `CLI-01`: 公共 CLI 不再暴露 `server` 命令
- [x] `CLI-02`: 公共 CLI 不再暴露 `watch` 命令
- [x] `CLI-03`: 公共 CLI 不再暴露 `report` 命令
- [x] `CLI-04`: 公共 CLI 不再暴露 `logs` 命令
- [x] `CLI-05`: CLI 注册、README、AI 文档、护栏脚本中的保留命令列表保持一致

### Analyze Contract

- [x] `ANL-01`: `analyze -i find` 提供符号 / 文件发现能力
- [x] `ANL-02`: `analyze -i read` 返回统一的复杂度 / 影响分析结构
- [x] `ANL-03`: `analyze -i link` 返回统一的引用 / 依赖关联结构
- [x] `ANL-04`: `analyze -i show` 覆盖展示 / 导出类分析结果
- [x] `ANL-05`: 旧 analyze intent 在两个 minor 版本内输出弃用警告，之后统一报 `E0001_INVALID_INTENT`

### Workflow & Automation

- [x] `FLOW-01`: `workflow` 的阶段只保留 `find → read → link → show`
- [x] `FLOW-02`: workflow 不再建模 `implementation`、`commit`、运行 CI 这类越界阶段
- [x] `CI-01`: `ci` 提供 `check-working-tree`
- [x] `CI-02`: `ci` 提供 `check-branch`
- [x] `CI-03`: `ci` 提供 `check-scripts`
- [x] `SHIP-01`: `ship` 的 check 阶段通过 `ci` 命令执行，而不是重复实现检查逻辑

### File Discovery & Documentation

- [x] `FILE-01`: 所有扫描类命令共享一个 `.gitignore` 感知的排除模块，并在无 `.gitignore` 时回退到内置默认规则
- [x] `DOC-01`: `README.md`、`AI_GUIDE.md`、`docs/ai-guide/COMMANDS.md`、`docs/ai-guide/OUTPUT.md` 与实现保持一致
- [x] `DOC-02`: 文档护栏脚本、CLI docs-sync 测试、CI 配置在重构后仍能通过
