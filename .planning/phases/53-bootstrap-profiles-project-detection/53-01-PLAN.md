Create `src/cli/init/detect.ts` with the following exact exports and behavior:

1. **Types:**
   ```typescript
   export type ProjectType = 'nodejs' | 'python' | 'go' | 'rust' | 'generic';
   export interface DetectionCandidate { type: ProjectType; markerFile: string; confidence: 'high' | 'low'; }
   export interface DetectionResult { candidates: DetectionCandidate[]; recommended: ProjectType | null; }
   ```

2. **Marker map (constant, frozen):**
   ```typescript
   const MARKER_MAP: Record<string, ProjectType> = {
     'package.json': 'nodejs',
     'pyproject.toml': 'python',
     'go.mod': 'go',
     'Cargo.toml': 'rust',
   };
   ```

3. **`detectProjectType(rootDir: string): DetectionResult`** — synchronous, uses `existsSync` from `node:fs`:
   - Iterate `Object.entries(MARKER_MAP)`.
   - For each marker file where `existsSync(path.join(rootDir, file))` is true, push `{ type, markerFile: file, confidence: 'high' }` to candidates.
   - If `candidates.length === 0`, return `{ candidates: [], recommended: null }`.
   - If `candidates.length === 1`, return `{ candidates, recommended: candidates[0].type }`.
   - If `candidates.length > 1`, return `{ candidates, recommended: null }` (forces disambiguation per D-02).

4. **`promptForProfileSelection(candidates: DetectionCandidate[]): Promise<ProjectType>`** — interactive readline prompt:
   - Import `createInterface` from `node:readline/promises` and `{ stdin as input, stdout as output }` from `node:process`.
   - Print `console.log('检测到多个项目类型标记：')`.
   - Print each candidate as `console.log(`  ${i + 1}) ${c.type} (${c.markerFile})`)`.
   - Create readline interface, ask `console.log('请选择 (输入编号): ')` (use `readline.question`).
   - Parse answer with `parseInt(answer.trim(), 10) - 1`.
   - If index is valid, return `candidates[index].type`; otherwise throw `new Error('无效选择')`.
   - Always `readline.close()` in a `finally` block.

5. **Security:** Do NOT read file contents — only check existence. Do NOT use `__dirname`.

6. **No `process.exit()` in this module.** Throw errors for invalid states; let the CLI entry handle exit.

Create `src/cli/init/profile-loader.ts` with the following exact behavior:

1. **Imports:** `z` from `zod`, `readFileSync` from `node:fs`, `path` from `node:path`, `fileURLToPath` from `node:url`.

2. **Schema (exact shape per D-08):**
   ```typescript
   export const bootstrapProfileSchema = z.object({
     parser: z.object({
       include: z.array(z.string()).min(1),
       extensions: z.array(z.string()).min(1),
     }),
     ignore: z.array(z.string()),
     analysis_depth: z.enum(['shallow', 'standard', 'deep']),
   });
   export type BootstrapProfile = z.infer<typeof bootstrapProfileSchema>;
   ```

3. **Allowed profile names (allow-list per security threat model):**
   ```typescript
   const ALLOWED_PROFILE_NAMES = new Set(['nodejs', 'python', 'go', 'rust', 'generic']);
   ```

4. **`resolvePackageRoot(): string`** — identical to hooks.ts pattern:
   ```typescript
   return fileURLToPath(new URL('../../../', import.meta.url));
   ```

5. **`resolveProfilePath(profileName: string): string`**:
   - If `!ALLOWED_PROFILE_NAMES.has(profileName)`, throw `new Error(`未知 profile: ${profileName}`)`.
   - If `profileName.includes('/') || profileName.includes('\\') || profileName.includes('..')`, throw `new Error(`非法 profile 名称: ${profileName}`)`.
   - Return `path.join(resolvePackageRoot(), 'src', 'cli', 'init', 'profiles', `${profileName}.json`)`.

6. **`loadProfile(profileName: string): BootstrapProfile`**:
   - Call `resolveProfilePath(profileName)` to get file path.
   - Read file with `readFileSync(filePath, 'utf8')`.
   - Parse with `JSON.parse(text)`.
   - Call `bootstrapProfileSchema.parse(parsed)` and return result.
   - Catch `ZodError` (imported from `zod`) and re-throw with a flattened human-readable message:
     ```typescript
     const issues = error.issues.map((issue) => `${issue.path.join('.')}: ${issue.message}`).join('; ');
     throw new Error(`Profile 验证失败 [${profileName}]: ${issues}`);
     ```
   - Catch `ENOENT` / generic read errors and re-throw as `new Error(`Profile 文件未找到 [${profileName}]: ${filePath}`)`.

7. **No `process.exit()` in this module.**
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI entry -> detection module | Untrusted filesystem state crosses into application logic |
| Profile name -> filesystem read | Untrusted string (from --profile flag) crosses into file read |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-01 | Tampering | `--profile ../../../etc/passwd` | mitigate | Allow-list profile names; reject names containing `/`, `\`, or `..` in `resolveProfilePath` |
| T-53-02 | Denial of Service | Malformed profile JSON causing crash | mitigate | zod validation catches malformed JSON; catch and format errors gracefully in `loadProfile` |
| T-53-03 | Denial of Service | Large profile JSON causing memory exhaustion | accept | Profile files are small (<1KB each); no user upload in v1 (D-05/D-06) |
| T-53-04 | Information Disclosure | ZodError leaks filesystem paths | mitigate | Error messages use relative profile name, not full file path, in production-facing output |
</threat_model>
