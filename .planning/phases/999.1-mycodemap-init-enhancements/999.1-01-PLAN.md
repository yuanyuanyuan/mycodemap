<objective>
Establish the canonical workspace/config foundation for Phase 999.1: shared path and config loading code must prefer `.mycodemap/config.json`, default fresh projects into `.mycodemap/`, and keep legacy root config files visible as migration inputs for `init` rather than hidden steady-state fallbacks.

Purpose: If `init` writes `.mycodemap/config.json` before the shared loaders understand that path, downstream commands will ignore the newly initialized project state.
Output: Updated path/config helpers plus focused tests proving canonical, legacy, and no-config behavior.
</objective>

<threat_model>
- Threat: `mycodemap init` creates `.mycodemap/config.json` but `generate`, `query`, or other commands still read root config paths
  - Severity: high
  - Mitigation: Update `resolveConfigPath()` / `loadCodemapConfig()` first and add tests where only `.mycodemap/config.json` exists
- Threat: Legacy `.codemap/storage` remains in fresh default config and recreates the old two-center workspace story
  - Severity: medium
  - Mitigation: Update default storage output and tests so fresh config points storage under `.mycodemap/storage`
</threat_model>
