<objective>
Refactor `mycodemap init` into a state reconciler with explicit asset statuses, deterministic receipt output, `--yes` / `--interactive` behavior, and first-run guidance that reflects the new two-step setup flow.

Purpose: Users and future agents must know which CodeMap assets were created, migrated, skipped, conflicted, or still need manual action.
Output: Reconciler helpers, receipt JSON under `.mycodemap/status/init-last.json`, updated terminal receipt, and tests for idempotent init behavior.
</objective>

<threat_model>
- Threat: Init prints a green success even when manual actions or conflicts remain
  - Severity: high
  - Mitigation: Receipt and terminal summary must separately list `done`, `manual-action-needed`, `conflict`, and `skipped` assets
- Threat: Large init refactor creates an overlong `initCommand` that violates repo redlines and becomes hard to test
  - Severity: medium
  - Mitigation: Move scan/apply/render logic into focused helpers with unit-tested pure functions
</threat_model>
