# Phase 81 Plan 01

## Objective

为 dependency / graph edge ID 建立单一 canonical normalization seam，并把 generation、SQLite writeback、legacy DB initialize backfill 与 query-facing tests 全部收口到同一规则，满足 `POL-03`。

## Tasks

1. 在领域层定义 canonical edge ID helper
   - verify: 相同语义的 module/symbol edge 生成同一个 lowercase / underscore-safe ID
2. 把 generate 与 SQLite normalizeGraph 接到同一 helper
   - verify: 新生成 graph 与写回 SQLite 后的 dependency ID 一致
3. 为 legacy SQLite truth 增加 initialize-time backfill rewrite
   - verify: 旧 edge ID 重新打开后会自动归一化，且 `dependencies` / `graph_edges` 仍保持 parity
4. 更新 focused tests
   - verify: storage / governance cache / query / MCP 断言统一读取 canonical edge ID，且覆盖至少一个 migration/failure 场景

## Acceptance

- 同一 graph semantics 在 repeated generate/save/load/query 路径上暴露相同 edge ID
- `dependencies.id` 与 `graph_edges.dependency_id` 始终保持 parity
- 旧 SQLite graph truth 在 initialize backfill 后不丢依赖语义，也不制造 duplicate artifact
