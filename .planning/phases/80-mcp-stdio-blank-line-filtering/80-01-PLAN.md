# Phase 80 Plan 01

## Objective

为 `mcp start` 的 stdio server path 增加 blank-line filtering，并给 malformed non-blank payload 补显式 parse failure，满足 `POL-02`。

## Tasks

1. 新建仓库自有 stdio transport seam
   - verify: 空白行在 parse 前被忽略；非空坏 payload 产生显式 JSON-RPC parse error
2. 将 `startCodeMapMcpServer` 切到该 transport
   - verify: 正常 MCP client 仍可 `listTools` / `callTool`
3. 增加真实 transport 测试
   - verify: blank-line noise 不破坏后续合法请求；malformed non-blank payload 不再 silent swallow

## Acceptance

- 默认 stdio MCP server path 不再把空白行当成待解析请求
- stdout 仍然只输出协议帧
- 至少一个失败测试证明 malformed non-blank payload 仍显式失败
