<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Raw dependency artifacts -> graph truth | Duplicate edges can look like stronger connectivity unless canonicalized before persistence |
| Persisted SQLite rows -> module projection | Repeated `graph_edges` rows can amplify weights if read/projection treats aggregation as dedup |
| Topology result -> MCP consumer | Raw scores without warning context can overclaim certainty on sparse or partial graph truth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-73-01 | Integrity | dependency identity | mitigate | Define one canonical duplicate key that reflects entity roles and dependency semantics before graph truth is persisted |
| T-73-02 | Integrity | SQLite writeback | mitigate | Collapse duplicates before writing `dependencies` and `graph_edges`, then verify row parity in tests |
| T-73-03 | Integrity | topology projection | mitigate | Dedup again before aggregation/scoring so stale persisted duplicates cannot inflate hub / bridge output |
| T-73-04 | Scope creep | topology feature design | mitigate | Keep first release module-level, interpretable, and on the existing community/MCP surface; defer symbol-level or schema-redesign work |
</threat_model>
