**Step 1: Create `src/cli/doctor/types.ts`** — Define the core diagnostic type contracts.

```typescript
// [META] since:2026-04-30 | owner:cli-team | stable:false
// [WHY] Shared diagnostic types for codemap doctor — single source of truth for checker output shape

export type DiagnosticCategory = 'install' | 'config' | 'runtime' | 'agent';
export type DiagnosticSeverity = 'ok' | 'warn' | 'error' | 'info';

export interface DiagnosticResult {
  category: DiagnosticCategory;
  severity: DiagnosticSeverity;
  id: string;             // English kebab-case, e.g. "ghost-command-detected"
  message: string;        // Human-readable description
  remediation: string;    // Plain text remediation instruction
  nextCommand?: string;   // Optional executable command for remediation
}

export interface DoctorOptions {
  json?: boolean;
  cwd?: string;
}

export interface DoctorReport {
  results: DiagnosticResult[];
  exitCode: number;  // 0=all pass, 1=has errors, 2=warnings only
}
```

Per CONTEXT.md decisions: severity levels are `ok`/`warn`/`error`/`info` (4-tier), diagnostic IDs are English kebab-case, remediation includes optional `nextCommand`.

**Step 2: Create `src/cli/doctor/check-ghost-commands.ts`** — Ghost command detection (per D-02, TRUST-01).

Implementation:
- Read `package.json` from `rootDir` using `readFileSync`
- Parse `scripts` object
- For each script value, check if it matches the pattern `echo '...not installed` or starts with `echo '...` followed by an install suggestion
- For each match, emit a `DiagnosticResult` with:
  - category: `'install'`
  - severity: `'error'`
  - id: `'ghost-command-detected'`
  - message: `Script "{scriptName}" is an echo stub, not a real command`
  - remediation: `Install the missing dependency or remove the script from package.json`
  - nextCommand: extract the suggested install command from the echo text (e.g., `npm i -D dependency-cruiser`)
- If no ghost commands found, emit a single `ok` result with id `'ghost-commands-ok'`
- Do NOT scan global node_modules — project-level package.json only (per CONTEXT.md deferred items)

**Step 3: Create `src/cli/doctor/check-native-deps.ts`** — Native dependency health (per D-02, TRUST-02).

Implementation:
- Reuse `detectTreeSitterSync()` from `src/cli/tree-sitter-check.ts` for tree-sitter detection

- Use `createRequire` from `node:module` to attempt CJS require of better-sqlite3 (it is a CJS native module)

  - If available: `DiagnosticResult` with severity `'ok'`, id `'{dep-name}-available'`
  - If not available: `DiagnosticResult` with severity `'error'`, id `'native-dep-missing'`, message describing which dep failed and the error, remediation suggesting `npm install` or `--wasm-fallback` (Phase 45 will make this actionable)
- Do NOT attempt to actually open a database — only verify the module can be imported/required

**Step 4: Create `src/cli/doctor/check-workspace-drift.ts`** — Workspace drift detection (per D-02, TRUST-03).

Implementation:
- Read `.mycodemap/status/init-last.json` from `rootDir` using `readFileSync`
- If file does not exist: emit `DiagnosticResult` with severity `'warn'`, id `'workspace-not-initialized'`, remediation: `Run 'codemap init' to create the workspace`
- Parse the receipt JSON as `InitReceipt` type (import from `../init/reconciler.js`)
- For each asset in `receipt.assets` that has a `path` field:
  - Check if the path exists on filesystem using `existsSync`
  - If missing: emit `DiagnosticResult` with severity `'warn'`, id `'workspace-drift-detected'`, message: `Receipt asset "{asset.key}" ({asset.label}) not found at {relativePath}`
  - For `tool-owned` assets: severity `'error'` (tool should manage these)
  - For `user-owned`/`team-owned` assets: severity `'warn'` (user may have intentionally removed)
- If all assets match: emit single `ok` result with id `'workspace-drift-ok'`

```typescript
// [META] since:2026-04-30 | owner:cli-team | stable:false
// [WHY] Orchestrate all doctor checkers and compute aggregate exit code per CONTEXT.md exit behavior decision
```

Implementation:
- Import all 4 checkers
- `runDoctor(options: DoctorOptions): Promise<DoctorReport>`:
  1. Determine `rootDir` from `options.cwd ?? process.cwd()`
  2. Run all checkers in parallel using `Promise.all`:
     - `checkGhostCommands(rootDir)` — sync, wrap in resolved Promise
     - `checkNativeDeps()` — sync, wrap in resolved Promise
     - `checkWorkspaceDrift(rootDir)` — sync, wrap in resolved Promise
     - `checkAgent()` — async
  3. Flatten all results into a single `DiagnosticResult[]`
  4. Compute exit code:
     - If any result has `severity === 'error'`: exitCode = 1
     - Else if any result has `severity === 'warn'`: exitCode = 2
     - Else: exitCode = 0
  5. Return `{ results, exitCode }`
