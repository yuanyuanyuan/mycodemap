| Boundary | Description |
|----------|-------------|
| Python AST call expression -> graph truth | Uncertain receiver shapes can look like valid edges; only provable calls may cross into shared truth |
| Import string -> cross-file target symbol | Module resolution can silently over-match if Python path handling is loose |
| Parser result -> analyzer module output | Explicit unresolved facts can be dropped if the shared contract is only partially extended |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-70-01 | Integrity | Python call extraction | mitigate | Emit only proven static edges and represent dynamic/ambiguous cases explicitly rather than guessing |
| T-70-02 | Integrity | method symbol identity | mitigate | Use stable qualified method symbols to prevent collisions between same-named methods in different classes |
| T-70-03 | Tampering | cross-file import resolution | mitigate | Resolve imported callees only from indexed imports/exports and exact Python file matches; avoid fuzzy scan-all heuristics |
| T-70-04 | Availability | parser/analyzer contract drift | mitigate | Extend the shared contract once and verify parser/global-index/analyzer seams together |
</threat_model>
