### Task 3: Extend validate-docs.js to detect npm script stubs

<read_first>
- scripts/validate-docs.js
- package.json
</read_first>

<action>
Add a new validation function `validateNpmScriptsAreReal` to scripts/validate-docs.js that:

1. Scans all markdown files in docs/ and .github/ for references to `npm run <script>`
2. For each referenced script, checks if it exists in package.json scripts
3. If the script exists, verifies it's NOT an echo stub (does not match `/^echo\s+['"].*not installed/i`)
4. Reports failures with file path and line context

The function should be called from the main validation flow alongside existing checks.

Note: Some `npm run` references in docs may be examples or instructions (like "run npm i -D dependency-cruiser"). Only check `npm run <scriptname>` patterns where `<scriptname>` is a script defined in package.json.

Implementation approach:
- Use regex `/npm\s+run\s+(\S+)/g` to find script references in doc files
- For each match, check `packageJson.scripts[scriptName]`
- If the script value matches the echo stub pattern, report as failure
- Skip references inside code blocks that are clearly examples (optional — can be v2)
</action>

<acceptance_criteria>
- `scripts/validate-docs.js` contains a function `validateNpmScriptsAreReal`
- The function is called during validation
- `npm run docs:check` passes (no existing real scripts are flagged as stubs)
- If a test echo stub is temporarily added to package.json, `npm run docs:check` fails
</acceptance_criteria>

### Task 4: Add validation decision tree to CLAUDE.md

<read_first>
- CLAUDE.md
- docs/rules/validation.md
</read_first>

<action>
Add a new "Validation Decision Tree" section to CLAUDE.md, near the existing validation router entry (around line 30). The section should be a concise 1-screen decision tree that helps developers choose the right validation path based on their change type.

Format:
```markdown
## Validation Decision Tree

| Change Type | First Check | Then Check | Gate |
|---|---|---|---|
| Docs only | `npm run docs:check` | — | Must pass before commit |
| Code + tests | `npm run check:all` | `npm run test:e2e` (if integration) | Must pass before PR |
| CLI command changes | `npm run build` + run the command | `npm run test` (relevant suites) | Must pass before PR |
| New dependency | `npm audit` | `npm run build` | Must pass before commit |
| CI/workflow changes | Check `.github/workflows/` syntax | Trigger test run | Must pass before merge |
| Release prep | `npm run docs:check:pre-release` | Manual smoke test | Must pass before tag |

**Quick reference:** All changes → `npm run check:all` is the comprehensive gate.
```

This should be placed after the router table or in a logical location within CLAUDE.md.
</action>
