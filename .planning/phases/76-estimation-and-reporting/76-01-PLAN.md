<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted detail rows -> grouped report summaries | Incorrect aggregation silently misstates token-cost reality for CI and humans |
| explicit `report` command -> report generation policy | Auto-running here would hide missing baseline data and violate the locked UX boundary |
| service truth -> human renderer | If renderer recomputes grouped stats independently, human and JSON outputs can drift |
| report contract -> downstream CI consumers | Dynamic or unstable grouped-summary fields create needless migration churn for later phases |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-76-01 | Integrity | grouped aggregation | mitigate | Compute grouped summaries in one service-layer helper and assert avg/min/max results in tests |
| T-76-02 | Reliability | explicit report mode | mitigate | Split root vs explicit report paths and test no-run behavior directly |
| T-76-03 | Drift | human vs JSON output | mitigate | Reuse one shared grouped report truth structure and keep renderer read-only |
| T-76-04 | Scope creep | reporting schema | mitigate | Exclude percentiles, thresholds, trends, and multi-run comparison from code and tests |
</threat_model>
