# Phase 85 Plan 01

## Objective

把 hook protocol 再向“纯 agent 控制协议”推进一步：压缩无关日志、补齐 protocol-only 模式和 log-path fallback，并把 checks 状态语义说清楚。

## Tasks

1. 增加 protocol-only 输出模式
   - verify: `CODEMAP_PROTOCOL_ONLY=1` 时不再输出人类 banner，只保留短协议行
2. 显式暴露 protocol log path
   - verify: 每次 hook 都输出 `CODEMAP_PRECHECK_LOG_PATH:`，log 文件内容可回放完整 JSON
3. 细化状态与 report-only 结果语义
   - verify: `not_applicable` / `skipped` / `warn` 分工明确，rule validation limit-reached 不再使用 `timed out`
4. 扩展 tests / smoke
   - verify: payload test 覆盖 protocol-only；smoke 覆盖 report-only limit-reached runtime case

## Acceptance

- 低噪音模式对 autonomous agent 友好，但默认 mixed 模式仍保留人类可读诊断
- 任何 protocol truncation 都至少能退回到 `log_path`
- 状态语义能直接驱动 agent 路由，不再逼迫其猜“没跑”还是“不适用”
