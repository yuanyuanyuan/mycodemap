<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| delegated-start hook input -> reminder engine | Raw runtime payload is untrusted and may omit, spoof, or reorder fields. |
| reminder engine -> silence ledger | Atomic first-seen writes must not race or leak between sessions/roles. |
| reminder adapter -> env-contract retrieval surface | Retrieval can fail or be unavailable; the hook must surface that failure visibly and continue. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-01 | Spoofing/Tampering | hook payload normalization | mitigate | Accept only the locked delegated-start events, normalize role/session fields explicitly, and reject unknown payload shapes with a visible warning instead of silently classifying them. |
| T-74-02 | Information Disclosure | session-role ledger | mitigate | Key silence only by parent session × role and keep the ledger temp/local so reminders do not leak across sessions or runtimes. |
| T-74-03 | Denial of Service | concurrent delegated starts | mitigate | Use atomic first-seen marker creation so parallel starts cannot duplicate reminders or corrupt silence state. |
| T-74-04 | Elevation / Policy bypass | retrieval failure path | mitigate | Never fall back to hidden prompt snippets or cached rules; emit a visible actionable warning and continue delegated work exactly as D-08 through D-10 require. |
</threat_model>
