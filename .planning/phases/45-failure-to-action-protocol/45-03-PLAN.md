# Phase 45 Plan 03: Migrate Priority Commands to ActionableError

## Objective

Migrate the 3 priority commands (`analyze`, `query`, `deps`) to produce `ActionableError` on failure. Each command's catch blocks should use `normalizeError(error, attempted)` to generate structured, actionable errors with appropriate codes, root causes, remediation plans, and nextCommand suggestions. Unmigrated commands benefit from the upgraded `normalizeError` automatically.

## must_haves

- [ ] `analyze` command catch blocks produce ActionableError via normalizeError
- [ ] `query` command catch blocks produce ActionableError via normalizeError
- [ ] `deps` command catch blocks produce ActionableError via normalizeError
- [ ] Each migrated command's ActionableError includes `attempted` description matching the operation
- [ ] formatError renders the full ActionableError output in both JSON and human modes
- [ ] Existing --json / --human output mode behavior preserved

## truths

- analyze, query, deps were already migrated to output mode in Phase 44 — we add ActionableError on top
- The `normalizeError` auto-upgrade from Plan 01 means even unmigrated commands get basic ActionableError fields
- `formatError` already handles ActionableError rendering from Plan 01
