# Phase 999.1: KùzuDB 作为主要存储 - Research

**Researched:** 2026-03-25
**Domain:** Graph Database Storage, KùzuDB Native Graph Model, Migration Strategy
**Confidence:** MEDIUM (KùzuDB archived status introduces significant uncertainty)

## Summary

本阶段研究如何将 KùzuDB 从当前的 snapshot 存储模式演进为原生图模型存储，并将其提升为默认主存储后端。研究涵盖了 KùzuDB 的技术特性、原生图建模方法、从文件系统的迁移策略、降级容错机制以及风险评估。

**关键发现：** KùzuDB 项目已于 2025 年 10 月 10 日被官方归档并停止维护。这是一个需要立即重新评估的重大风险因素。虽然现有功能仍可正常使用，但长期维护性和社区支持存在不确定性。

**Primary recommendation:** 在继续推进 KùzuDB 作为主存储的同时，必须并行评估替代方案（FalkorDB、DuckPGQ）并制定迁移预案。

### Pattern 1: Native Graph Model Schema
**What:** 使用 KùzuDB 原生节点表和关系表存储图数据
**When to use:** 替代当前 snapshot JSON 序列化模式

**Schema Design:**
```cypher
// 节点表定义
CREATE NODE TABLE Module(
  id STRING PRIMARY KEY,
  projectId STRING,
  path STRING,
  language STRING,
  lines INT64,
  codeLines INT64,
  commentLines INT64,
  blankLines INT64
);

CREATE NODE TABLE Symbol(
  id STRING PRIMARY KEY,
  moduleId STRING,
  name STRING,
  kind STRING,
  visibility STRING,
  file STRING,
  line INT64,
  column INT64
);

CREATE NODE TABLE File(
  id STRING PRIMARY KEY,
  path STRING,
  lastModified STRING
);

// 关系表定义
CREATE REL TABLE IMPORTS(FROM Module TO Module);
CREATE REL TABLE EXPORTS(FROM Module TO Symbol);
CREATE REL TABLE DEPENDS_ON(FROM Symbol TO Symbol);
CREATE REL TABLE CONTAINS(FROM File TO Module);
CREATE REL TABLE CALLS(FROM Symbol TO Symbol);
```

### Pattern 2: Storage Fallback Chain
**What:** KùzuDB 初始化失败时自动降级到文件系统
**When to use:** 保证系统可用性， graceful degradation

**Implementation:**
```typescript
// StorageFactory.ts
async determineStorageType(config: StorageConfig): Promise<StorageType> {
  if (config.type !== 'auto' && config.type !== 'kuzudb') {
    return config.type;
  }

  // 优先尝试 KùzuDB
  const kuzuAvailable = await this.checkKuzuAvailability();
  if (kuzuAvailable) {
    return 'kuzudb';
  }

  // Fallback 到文件系统
  if (config.type === 'kuzudb') {
    logger.warn('KùzuDB unavailable, falling back to filesystem');
  }
  return 'filesystem';
}

private async checkKuzuAvailability(): Promise<boolean> {
  try {
    const kuzu = await import('kuzu');
    // 尝试创建临时数据库验证功能正常
    const tempDb = new kuzu.Database(':memory:');
    await tempDb.close();
    return true;
  } catch {
    return false;
  }
}
```

### Pattern 3: Data Migration with Backup
**What:** 文件系统数据自动迁移到 KùzuDB，保留原数据备份
**When to use:** 用户首次切换到 KùzuDB 时

**Implementation:**
```typescript
// KuzuDBStorage.ts
async migrateFromFilesystem(fsStorage: FileSystemStorage): Promise<boolean> {
  const existingData = await fsStorage.loadCodeGraph();

  if (!existingData || existingData.modules.length === 0) {
    return false; // 无数据需要迁移
  }

  // 检查 KùzuDB 是否已有数据
  const kuzuData = await this.loadCodeGraph();
  if (kuzuData.modules.length > 0) {
    logger.info('KùzuDB already contains data, skipping migration');
    return false;
  }

  // 执行迁移
  await this.saveCodeGraph(existingData);

  // 备份原文件
  await fsStorage.backup();
  logger.info(`Migrated ${existingData.modules.length} modules to KùzuDB`);

  return true;
}
```

### Anti-Patterns to Avoid
- **Snapshot Anti-Pattern:** 继续将整个图序列化为单个 JSON 字符串存储，失去图数据库优势
- **Dual-Write Anti-Pattern:** 同时写入文件系统和 KùzuDB，增加复杂性和不一致风险
- **Silent Fallback:** 降级时不记录日志，导致用户无法察觉存储后端变化

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| 图遍历查询 | 自定义内存遍历算法 | KùzuDB Cypher 查询 | 向量化和因子化查询处理器，多核并行 |
| 事务管理 | 自定义 ACID 实现 | KùzuDB 内置事务 | 可序列化 ACID，已验证正确性 |
| 数据序列化 | 自定义二进制格式 | KùzuDB 列式存储 | CSR 邻接表索引，高效压缩 |
| 并发控制 | 自定义锁机制 | KùzuDB 连接池 | 嵌入式数据库自动处理并发 |

**Key insight:** KùzuDB 的核心价值在于其列式存储和向量化查询引擎，自定义实现无法达到同等性能。

## Common Pitfalls

### Pitfall 1: KùzuDB Project Abandonment
**What goes wrong:** KùzuDB 已于 2025 年 10 月 10 日被归档，不再维护
**Why it happens:** Kùzu Inc. 放弃项目转向其他工作
**How to avoid:**
- 短期：继续使用 v0.11.3，监控社区 fork（LadybugDB, Bighorn）
- 中期：评估 FalkorDB 或 DuckPGQ 作为替代
- 长期：制定 6-12 个月迁移计划
**Warning signs:** 安全漏洞无修复、Node.js 新版本不兼容、依赖冲突

### Pitfall 2: Native Binding Failures
**What goes wrong:** KùzuDB 依赖原生 Node.js 绑定，在特定平台或 Node 版本上可能失败
**Why it happens:** 预编译二进制不支持所有平台组合
**How to avoid:**
- 实现可靠的 fallback 机制
- 在 CI 中测试多平台安装
- 提供清晰的错误信息和解决指南
**Warning signs:** `Error: Cannot find module 'kuzu'`、`kuzu native binding unavailable`

### Pitfall 3: Schema Migration Complexity
**What goes wrong:** 原生图模型 schema 演进时需要数据迁移
**Why it happens:** KùzuDB 不支持在线 schema 变更
**How to avoid:**
- 实现 schema 版本管理
- 提供数据导出/导入工具
- 保留向后兼容的读取能力
**Warning signs:** 查询失败、数据丢失、schema 不匹配错误

### Pitfall 4: Memory Pressure on Large Graphs
**What goes wrong:** 大型代码库的图数据可能超出内存限制
**Why it happens:** KùzuDB 是内存映射数据库，大图需要足够 RAM
**How to avoid:**
- 监控内存使用
- 实现分页加载
- 考虑分项目存储
**Warning signs:** 性能下降、OOM 错误、系统交换增加

## Code Examples

### Native Graph Model Implementation
```typescript
// KuzuDBStorage.ts - 原生图模型实现示例
export class KuzuDBStorage extends StorageBase {
  private async initializeSchema(): Promise<void> {
    // 创建节点表
    await this.runQuery(`
      CREATE NODE TABLE IF NOT EXISTS Module(
        id STRING PRIMARY KEY,
        projectId STRING,
        path STRING,
        language STRING,
        lines INT64,
        codeLines INT64,
        commentLines INT64,
        blankLines INT64
      )
    `);

    await this.runQuery(`
      CREATE NODE TABLE IF NOT EXISTS Symbol(
        id STRING PRIMARY KEY,
        moduleId STRING,
        name STRING,
        kind STRING,
        visibility STRING,
        file STRING,
        line INT64,
        column INT64
      )
    `);

    // 创建关系表
    await this.runQuery(`
      CREATE REL TABLE IF NOT EXISTS IMPORTS(FROM Module TO Module)
    `);

    await this.runQuery(`
      CREATE REL TABLE IF NOT EXISTS CALLS(FROM Symbol TO Symbol)
    `);
  }

  async saveCodeGraph(graph: CodeGraph): Promise<void> {
    // 清空现有数据
    await this.runQuery('MATCH (m:Module) DELETE m');
    await this.runQuery('MATCH (s:Symbol) DELETE s');
    await this.runQuery('MATCH ()-[r:IMPORTS]->() DELETE r');
    await this.runQuery('MATCH ()-[r:CALLS]->() DELETE r');

    // 批量插入模块
    for (const module of graph.modules) {
      await this.executePrepared(`
        CREATE (m:Module {
          id: $id,
          projectId: $projectId,
          path: $path,
          language: $language,
          lines: $lines,
          codeLines: $codeLines,
          commentLines: $commentLines,
          blankLines: $blankLines
        })
      `, {
        id: module.id,
        projectId: module.projectId,
        path: module.path,
        language: module.language,
        lines: module.stats.lines,
        codeLines: module.stats.codeLines,
        commentLines: module.stats.commentLines,
        blankLines: module.stats.blankLines,
      });
    }

    // 批量插入符号和关系...
  }

  async findCallers(functionId: string): Promise<Symbol[]> {
    const rows = await this.queryRows(`
      MATCH (caller:Symbol)-[:CALLS]->(callee:Symbol {id: $functionId})
      RETURN caller.id AS id, caller.name AS name, caller.kind AS kind
    `, { functionId });

    return rows.map(row => ({
      id: String(row.id),
      name: String(row.name),
      kind: String(row.kind),
      // ... 其他字段
    })) as Symbol[];
  }
}
```

### Fallback Storage Wrapper
```typescript
// FallbackStorage.ts
export class FallbackStorage implements IStorage {
  readonly type = 'filesystem' as const;
  readonly isFallback = true;
  readonly fallbackReason: string;

  constructor(
    private readonly delegate: FileSystemStorage,
    reason: string
  ) {
    this.fallbackReason = reason;
  }

  async initialize(projectPath: string): Promise<void> {
    logger.warn(`Using fallback storage: ${this.fallbackReason}`);
    return this.delegate.initialize(projectPath);
  }

  // 委托所有操作到文件系统存储...
  async saveCodeGraph(graph: CodeGraph): Promise<void> {
    return this.delegate.saveCodeGraph(graph);
  }
}
```
