<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted report rows -> gate verdict | If the gate reads grouped summaries or totals instead of rows, outlier regressions can be silently missed |
| shared report truth -> human/JSON renderer | If verdict data is recomputed in multiple places, outputs and exit code can drift |
| CLI threshold flag -> report/root path selection | If thresholding leaks into `token` or explicit `report` auto-runs, the command-family contract regresses |
| verdict -> exit code | If blocking behavior is applied before rendering or below the CLI edge, failures become harder to test and reason about |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-77-01 | Integrity | threshold truth source | mitigate | Compute gate verdicts only from `rows[*].estimatedTotalTokens` and pin grouped-average false-pass cases in tests |
| T-77-02 | Reliability | explicit report path | mitigate | Preserve `requireLatestReport()` behavior and test missing-run remediation under thresholded report mode |
| T-77-03 | Drift | human/JSON/exit parity | mitigate | Attach one canonical `gate` block to report truth and assert stdout JSON plus `process.exitCode` together |
| T-77-04 | Scope creep | command-family boundary | mitigate | Exclude `token` flag support, storage/schema changes, and Phase 78 analytics from code and tests |
| T-77-05 | Tampering | invalid threshold input | mitigate | Reject non-finite or negative threshold values at the command edge before gate evaluation |
</threat_model>
