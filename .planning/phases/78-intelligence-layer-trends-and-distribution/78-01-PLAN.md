<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted run metadata -> trend truth | If the service reads the wrong run ordering, latest-vs-previous deltas become false while still looking plausible |
| historical query-type rows -> percentile summaries | If percentile sample pools are implicit or unstable, `p50/p95/max` become noisy vanity metrics |
| shared report truth -> human/JSON renderer | If renderers recompute analytics, human and machine output drift |
| advisory intelligence -> Phase 77 gate | If new ranked fields leak into gate evaluation, the milestone silently creates a second blocking policy |
| historical reads -> CLI latency | If the phase scans unbounded data carelessly, report mode can regress into a slow analytics command |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-78-01 | Integrity | recent-run ordering | mitigate | Add explicit storage tests for `recorded_at DESC, id DESC` ordering and one-run history behavior |
| T-78-02 | Integrity | missing baseline | mitigate | Emit `baselineAvailable=false` and `previous*=null` when there is no previous run; assert this in service and command tests |
| T-78-03 | Drift | percentile math | mitigate | Keep nearest-rank helpers in `AgentMetricsService` and pin exact `p50/p95` outputs in tests |
| T-78-04 | Drift | renderer vs JSON contract | mitigate | Compute intelligence fields once in the service and expose them via additive contract assertions plus human/JSON command tests |
| T-78-05 | Scope creep | gate semantics | mitigate | Keep `gate` evaluation unchanged, assert unchanged threshold behavior in command tests, and prohibit new CLI flags |
| T-78-06 | Availability | historical sample reads | mitigate | Limit trend reads to 2 runs, read history by `queryType` only for keys present in the latest run, and surface sample counts explicitly |
</threat_model>
