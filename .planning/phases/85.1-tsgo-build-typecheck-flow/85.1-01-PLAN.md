# Phase 85.1 Plan 01

## Objective

把当前已存在的 TypeScript toolchain 改动收敛成一个可验证的最小 follow-up：统一 `build` / `typecheck` 到 `tsgo`，让 CLI entry 的 JSON import 语法适配当前工具链，并确保 Claude 的 TS 编辑后检查不再和 `package.json` 脚本漂移。

## Tasks

1. 固化当前 toolchain truth
   - verify: `package.json` 的 `build` / `typecheck` script 与 `.claude/settings.json` 使用同一条 `tsgo` 路径
2. 固化 lockfile 与依赖变更
   - verify: `package-lock.json` 锁定 `@typescript/native-preview` 及对应平台包
3. 验证 CLI entry import 语法
   - verify: `src/cli-new/index.ts` 的 `with { type: 'json' }` 不阻断 `npm run build`
4. 做第一轮真实执行验证
   - verify: `npm run build` 与 `npm run typecheck` 在当前工作区执行，记录成功或失败原因
5. 模拟失败场景并记录风险
   - verify: 若 `tsgo` 对 JSON import 或项目配置不兼容，文档中明确记录失败点与下一刀

## Acceptance

- `build` / `typecheck` / Claude PostToolUse 三者对 TypeScript 检查工具链的真相一致
- 至少有一轮真实命令执行证据，而不是只记录静态 diff
- 若验证失败，失败点被精确收敛到具体命令和报错，而不是继续模糊归因
