# Plan 01: Loader & Error Handler Wiring

## Objective
修复 v2.0 audit 中 3 个 critical blockers：
1. `validateTreeSitter()` 在 loader 之前抛出，阻断 WASM fallback 路径
2. `tryApplySuggestion()` 有完整实现但零调用点
3. `--wasm-fallback` / `--apply-suggestion` global flags 注册但从未消费

## must_haves
- `wrapWithTreeSitterCheck` 调用 `loadTreeSitter()` 而非 `validateTreeSitter()`，native 失败时自动走 WASM fallback
- CLI 错误处理路径统一调用 `tryApplySuggestion()`，读取 `cliOpts.applySuggestion`
- `--wasm-fallback` flag 被读取并在 loader 调用前激活 WASM 路径
- 所有修改保持现有命令行为不变（无 breaking change）

修改 `src/cli/index.ts`：

1. 在文件顶部添加 import：
   ```typescript

   ```

2. 新增集中式错误处理包装函数：
   ```typescript
   async function wrapWithErrorHandler<T>(
     commandName: string,
     action: (options: T) => Promise<void>
   ): Promise<(options: T) => Promise<void>> {
     return async (options: T) => {
       try {
         await action(options);
       } catch (error) {
         const mode = resolveOutputMode({});
         const formatted = formatError(error, mode, commandName);
         process.stdout.write(formatted + '\n');

         // 尝试自动修复（如果 --apply-suggestion 已设置且错误是 ActionableError）
         const cliOpts = program.opts();
         if (cliOpts.applySuggestion && error && typeof error === 'object' && 'nextCommand' in error) {
           const suggestionResult = await tryApplySuggestion(
             error as ActionableError,
             { applySuggestion: cliOpts.applySuggestion, wasmFallback: cliOpts.wasmFallback },
             mode
           );
           if (suggestionResult.success) {
             process.exitCode = 0;
             return;
           }
         }

         process.exitCode = 1;
       }
     };
   }
   ```

3. 在 `run()` 函数（或 `program.parse()` 之后）添加全局未捕获异常处理：
   ```typescript
   program.hook('postAction', async (thisCommand, actionCommand) => {
     // postAction 不接收错误，需要通过其他方式
     // 实际上 commander 的 error 事件更适合
   });
   ```

   改为使用 `program.exitOverride()` 或监听错误事件：
   ```typescript
   // 在 program.parse() 之前添加
   program.on('error', async (error) => {
     const cliOpts = program.opts();
     if (cliOpts.applySuggestion && error && typeof error === 'object' && 'nextCommand' in error) {
       const mode = resolveOutputMode({});
       const suggestionResult = await tryApplySuggestion(
         error as ActionableError,
         { applySuggestion: cliOpts.applySuggestion, wasmFallback: cliOpts.wasmFallback },
         mode
       );
       if (suggestionResult.success) {
         process.exitCode = 0;
         return;
       }
     }
   });
   ```

   实际上，更简单的方式是在每个 command action 中使用统一的包装。修改每个 `.action()` 调用：

   例如，对于 generate command：
   ```typescript
   .action(await wrapWithErrorHandler('generate', async (options, command: Command) => {
     await generateCommand({ ... });
   }));
   ```

   但这样改动量很大。更好的方式是创建一个统一的 action wrapper，同时包含 tree-sitter check 和 error handling：

   ```typescript
   async function createActionHandler<T>(
     commandName: string,
     action: (options: T) => Promise<void>
   ): Promise<(options: T) => Promise<void>> {
     return async (options: T) => {
       try {
         if (commandRequiresTreeSitter(commandName)) {
           await validateTreeSitterAsync();
         }
         await action(options);
       } catch (error) {
         const mode = resolveOutputMode({});
         const formatted = formatError(error, mode, commandName);
         process.stdout.write(formatted + '\n');

         const cliOpts = program.opts();
         if (cliOpts.applySuggestion && error && typeof error === 'object' && 'nextCommand' in error) {
           const suggestionResult = await tryApplySuggestion(
             error as ActionableError,
             { applySuggestion: cliOpts.applySuggestion, wasmFallback: cliOpts.wasmFallback },
             mode
           );
           if (suggestionResult.success) {
             process.exitCode = 0;
             return;
           }
         }

         process.exitCode = 1;
       }
     };
   }
   ```

   然后用 `createActionHandler` 替换所有 `wrapWithTreeSitterCheck` 的调用。

   需要为 `createActionHandler` 添加 ActionableError 的 import：
   ```typescript
   import type { ActionableError } from './output/types.js';
   ```

   注意：不是所有命令都需要 tree-sitter check。`createActionHandler` 只在 `commandRequiresTreeSitter` 返回 true 时才调用 `validateTreeSitterAsync`。
</action>

<action>
修改 `src/cli/index.ts`：

1. 在 `--native` flag 处理代码（line 243-247）之后、`program.parse()` 之前，添加 `--wasm-fallback` flag 的处理：

   ```typescript
   // Handle --wasm-fallback flag: activate WASM fallback before any command runs
   const cliOpts = program.opts();
   if (cliOpts.wasmFallback) {
     process.env.CODEMAP_USE_WASM_TREE_SITTER = '1';
     process.env.CODEMAP_USE_WASM_BETTER_SQLITE3 = '1';
   }
   ```

   注意：这段代码已经在 line 243 有 `const cliOpts = program.opts();`，所以 `--wasm-fallback` 的处理可以直接接在后面：

   ```typescript
   // Handle --native flag: disable WASM fallback before any command runs
   const cliOpts = program.opts();
   if (cliOpts.native) {
     process.env.CODEMAP_USE_WASM_TREE_SITTER = '0';
     process.env.CODEMAP_USE_WASM_BETTER_SQLITE3 = '0';
   }
   if (cliOpts.wasmFallback) {
     process.env.CODEMAP_USE_WASM_TREE_SITTER = '1';
     process.env.CODEMAP_USE_WASM_BETTER_SQLITE3 = '1';
   }
   ```

   实际上，当前的代码结构是：
   - line 96-98: flags 注册在 program 上
   - line 243-247: `cliOpts.native` 检查在 `program.parse()` 之后

   但 `program.parse()` 会解析参数，`program.opts()` 在 parse 之后才能获取到值。所以当前代码结构是正确的。

   只需要在 line 247 之后添加 wasmFallback 的处理即可。
