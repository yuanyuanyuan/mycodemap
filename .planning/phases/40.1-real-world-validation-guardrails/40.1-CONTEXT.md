Add mandatory real-world validation rules to testing docs, pre-release checklist, AGENTS.md, CLAUDE.md, and CI guardrails. All tests must include real test cases with evidence from actual execution. Reject any development/testing without real-world scenario validation. This is a governance/rules phase with zero runtime code changes (docs + CI only).

<decisions>
## Implementation Decisions

### 强制程度与阈值
- **D-01:** 真实场景验证为**硬约束** — AGENTS.md Section 8.1 声明阈值（真实 filesystem + 真实 subprocess/transport），CI 护栏作为执行控制机制。宪法先声明意图，CI 负责执行控制。分级落地：pre-commit warn-only → CI blocking → pre-release 必过。

### 自动化检查策略
- **D-02:** Pre-commit 用启发式 grep 检查测试文件中是否包含真实调用模式（`spawn`/`execSync`/`mktemp`/`InMemoryTransport`/`createServer`/`listen` 等）。Warn-only，不阻断。位置：在 "File headers passed" 之后、"Documentation guardrails" 之前。
- **D-03:** CI 用 `[证据]` 标签检查 PR commit messages / PR body。Warn-only 初始阶段，可渐进升级为 blocking。仅对 FEATURE/BUGFIX commit 类型触发。

### Phase 41/42 豁免边界
- **D-04:** 分层豁免 — Phase 41/42 已有实现不追溯合规，但后续任何修改必须合规。具体：
  - Phase 42 测试已使用真实 MCP transport（`StdioServerTransport` + `PassThrough`），符合阈值，只需补 `[证据]` 标签
  - Phase 41 测试仅为结构断言，后续修改时需补真实 CLI 调用测试
  - Phase 43+ 从 inception 起必须合规

### 文档增强策略
- **D-05:** 对 testing.md 和 pre-release-checklist.md 采用"加元信息层"策略 — 在现有段落头部加适用范围（Phase 41+ 新开发及后续修改）、豁免条款（41/42 已有实现不追溯）、交叉引用（→ AGENTS.md Section 8.1）。现有规则正文不变。

### Claude's Discretion
- Pre-commit grep 模式列表的具体词汇
- CI evidence 检查脚本的详细实现
- AGENTS.md Section 8.1/8.2 的具体措辞
- CLAUDE.md 路由标注的具体格式

</decisions>
