# Plan 47-01: WASM-First Native Dependency Wrappers

## Objective
Create thin wrapper modules that read `CODEMAP_USE_WASM_*` env vars (set by Phase 45's `checkAndActivateWasmFallback`) and dynamically import the correct implementation — native or WASM fallback. Add `--native` CLI flag to force native binary usage.

## Background
Phase 45 implemented `checkAndActivateWasmFallback()` which sets env vars like `CODEMAP_USE_WASM_TREE_SITTER=1` when native deps fail. However, the actual import sites (`tree-sitter-parser.ts`, `SQLiteStorage.ts`) still unconditionally import native modules. This plan bridges that gap.
