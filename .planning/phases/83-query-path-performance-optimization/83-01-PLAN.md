# Phase 83 Plan 01

## Objective

用最小改动把高频 graph read path 的重复全图扫描压掉：eager cache 预建并复用读索引，impact 查询复用有界结果缓存，handler 不再做第二次图遍历。

## Tasks

1. 预建 graph read index
   - verify: dependency lookup 与 impact helper 可以复用同一份内存索引而不改默认语义
2. 给 eager governance cache 加 bounded impact reuse
   - verify: eager 模式重复 impact 查询命中缓存，sqlite-direct fallback 仍走 SQLite truth
3. 去掉 QueryHandler 的二次 BFS
   - verify: `analyzeImpact()` 仅基于 storage 返回的 shared impact truth 投影旧 response shape
4. 补 focused regressions
   - verify: helper parity、handler no-second-walk、eager/direct parity 都保持绿色

## Acceptance

- eager governance cache 会复用 source/target adjacency index，而不是每次重新全扫 `dependencies`
- repeated module impact queries 在 eager 模式下会复用 bounded in-memory cache
- `QueryHandler.analyzeImpact()` 不再额外 `loadCodeGraph()` 或重新 BFS，但对外返回 shape 不变
- 相关 focused tests 与 `tsc --noEmit` 通过
