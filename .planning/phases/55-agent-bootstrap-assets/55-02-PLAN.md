3. **`createAssistantPlan(rootDir: string, assistantProfile?: AssistantProfile, profileName?: string): AssistantPlan` function:**

The `assistantProfile` parameter filters which assets to generate. If undefined, generate ALL four assets (per D-04).

Define a local `ASSISTANT_ASSETS` array with four entries, each having:
- `runtime: AssistantProfile` — which runtime this asset belongs to
- `fileName: string` — target file name
- `generate: (rootDir: string, profileName?: string) => string` — content generator

**Implementation logic for `createAssistantPlan`:**

For each asset in ASSISTANT_ASSETS:
- Filter: if `assistantProfile` is set and doesn't match `asset.runtime`, skip.
- Determine target path: `path.join(rootDir, '.mycodemap', 'assistants', asset.fileName)`
- Generate content: `asset.generate(rootDir, profileName)`
- Check if file already exists:
  - If exists and content matches: push `already-synced` asset, no write.
  - If exists but different: push `conflict` asset with `manualAction: '手动审阅 .mycodemap/assistants/{fileName} 的内容，确认后可删除该文件再重跑 init'`, no write.
  - If not exists: push `installed` asset + FileWriteAction.

Asset keys: `assistant:claude-context`, `assistant:agents-context`, `assistant:claude-hook-example`, `assistant:codex-agent-example`.
Ownership: `'tool-owned'`. Origin: `'assistant-bootstrap'`.
