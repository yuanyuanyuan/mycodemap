## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Python AST complexity extraction -> shared truth | A parser-local metric can drift from downstream truth if it is not written through the shared contract |
| Function detail -> symbol writeback | Name mismatches can attach complexity to the wrong symbol or silently drop it |
| Shared module truth -> CLI output | CLI fallback/recompute behavior can mask whether persisted truth is actually being consumed |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-72-02 | Integrity | symbol complexity backfill | mitigate | Match function/method details to stable symbol identities and test top-level plus qualified method cases |
| T-72-03 | Integrity | CLI proof path | mitigate | Add tests that distinguish persisted-read behavior from recompute behavior and keep recompute opt-in |
| T-72-04 | Scope creep | complexity implementation | mitigate | Defer cross-language complexity unification, preview output changes, and extra consumers beyond the chosen CLI proof path |
</threat_model>
