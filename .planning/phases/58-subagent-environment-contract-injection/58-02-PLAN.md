# Plan 58-02: CLI Retrieval Command, Interface Contract, and MCP Tool

<objective>
Expose the Project Environment Contract through `mycodemap env-contract`, register the command in the interface contract, and make `codemap_env_contract` available to subagents through MCP with a real contract payload.
</objective>

<threat_model>
T-58-05 broken subagent retrieval: a subagent may receive only a CLI redirect instead of the contract payload. Mitigation: test MCP `callTool()` for structured contract JSON.
T-58-06 invalid MCP tool name: hyphenated command names can produce unstable tool names. Mitigation: normalize to `codemap_env_contract` and assert `listTools()`.
T-58-07 accidental side effects: read-only retrieval should not rewrite `.mycodemap/env-contract.json` unless `--update` is passed. Mitigation: CLI tests compare file mtime/content.
</threat_model>

<tasks>

<action>
Create `src/cli/commands/env-contract.ts` exporting `envContractCommand`.

Command options:

```typescript
.option('-j, --json', 'Output contract as JSON')
.option('--human', 'Force human-readable output')
.option('--for <agentType>', 'Filter contract items for an agent type')
.option('--category <category>', 'Filter by contract category')
.option('--check', 'Check contract freshness and critical coverage')
.option('--update', 'Regenerate .mycodemap/env-contract.json')
.option('--as-hook-config', 'Print Claude Code SubagentStart hook example')
.option('--as-codex-agent', 'Print Codex agent developer_instructions example')
```

Behavior:
- Default read path loads `.mycodemap/env-contract.json` if present; if missing, discover from current files and return a contract without writing unless `--update` is set.
- `--update` writes `.mycodemap/env-contract.json`.
- `--for <agentType>` applies `filterContractForAgent()`.
- `--category <category>` filters by one category after agent filtering.
- `--check` runs the check function and sets `process.exitCode` to `0` for ok, `2` for warning-only conflicts, `1` for schema/missing critical/stale source errors.
- `--as-hook-config` prints a Claude Code hook example whose `additionalContext` tells subagents to run `mycodemap env-contract --for <type> --json`.
- `--as-codex-agent` prints TOML text whose `developer_instructions` tells Codex agents to call `mycodemap env-contract --for worker --json` or MCP `codemap_env_contract`.

Register it in `src/cli/index.ts` with `program.addCommand(envContractCommand)`.
</action>
