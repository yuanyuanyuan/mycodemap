# Plan 58-03: Init Assistant Guidance and Doctor Drift Diagnostics

<objective>
Update generated assistant assets so subagents retrieve project rules through CLI/MCP, and integrate env-contract freshness/conflict diagnostics into `mycodemap doctor`.
</objective>

<threat_model>
T-58-10 stale assistant guidance: generated examples may keep telling agents to `cat` the JSON file and bypass filtering. Mitigation: tests reject raw `cat .mycodemap/env-contract.json` guidance.
T-58-11 noisy doctor failures: conflicts should warn without blocking normal doctor use, but drift/missing critical contracts must be visible as errors. Mitigation: severity tests for both cases.
T-58-12 user file mutation: init must not auto-rewrite team-owned `CLAUDE.md`, `AGENTS.md`, `.claude/settings.json`, or `.codex/agents/*`. Mitigation: only generated examples under `.mycodemap/assistants/` are changed.
</threat_model>

### Task 58-03-03: Add env-contract doctor checker

<read_first>
- `src/cli/doctor/check-agent.ts` - checker style
- `src/cli/doctor/types.ts` - DiagnosticResult shape
- `src/cli/doctor/orchestrator.ts` - checker aggregation and exit code behavior

- `src/cli/env-contract/storage.ts` - contract load/write helpers
- `.planning/phases/58-subagent-environment-contract-injection/58-CONTEXT.md` - D-07 conflict behavior
</read_first>

<action>
Create `src/cli/doctor/check-env-contract.ts`:

```typescript
export function checkEnvContract(rootDir: string): DiagnosticResult[]
```

Map env-contract check results into `DiagnosticResult`:
- valid schema -> `severity: 'ok'`, `id: 'env-contract-schema-ok'`
- missing `.mycodemap/env-contract.json` -> `severity: 'warn'`, `id: 'env-contract-missing'`, remediation `Run mycodemap init or mycodemap env-contract --update`
- missing critical item -> `severity: 'error'`, `id: 'env-contract-critical-missing'`
- stale source snapshot -> `severity: 'error'`, `id: 'env-contract-source-drift'`
- conflicts only -> `severity: 'warn'`, `id: 'env-contract-conflict'`
- fresh/no conflicts -> `severity: 'ok'`, `id: 'env-contract-fresh'`

Add this checker to `runDoctor()`:

```typescript
Promise.resolve(checkEnvContract(rootDir))
```

Keep aggregate exit code logic unchanged: any `error` yields `1`, warnings only yield `2`.
</action>

</verify>

<acceptance_criteria>

- `runDoctor()` includes `checkEnvContract(rootDir)`.
- Missing contract produces `category: 'agent'` and `id: 'env-contract-missing'`.
- Stale source snapshot produces `severity: 'error'`.
- Conflict-only contract produces `severity: 'warn'`.
- Doctor exit code remains `2` for warning-only conflicts and `1` for stale/missing critical errors.
