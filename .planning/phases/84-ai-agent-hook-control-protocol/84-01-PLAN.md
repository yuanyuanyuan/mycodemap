# Phase 84 Plan 01

## Objective

把 git commit hooks 从“对人类有结构的阻断提示”升级为“对 AI Agent 可直接执行的 fail-fast 控制协议”，并确保 installable templates 与 managed copies 一致。

## Tasks

1. 前置 cheap blockers 并固化 pre-commit protocol
   - verify: `staged-file-limit` 在重型测试前阻断，且 `checks[0]` / `next_action=split_commit` 稳定
2. 给 commit-msg 补 protocol parity
   - verify: `commit-format` 与 `commit-scope-message` 都输出 `codemap.commitmsg.v1` 与 `rewrite_commit_message`
3. 保持 framework-agnostic remediation
   - verify: template 仍包含 `package-test` / `pytest` / `go test ./...` / `cargo test` generic fallback，并输出结构化 `verify_commands`
4. 补 contract + smoke coverage
   - verify: payload tests、workflow static checks、真实 `git commit` smoke 全绿
5. 把这次 hardening 登记为 between-milestones special phase
   - verify: `STATE` / `ROADMAP` / `PROJECT` / `MILESTONES` 都记录 Phase 84，但当前 milestone 仍为 `none`

## Acceptance

- `pre-commit` 与 `commit-msg` 都输出稳定 `CODEMAP_PRECHECK_PROTOCOL:<json>`
- protocol 自带 rule metadata、structured resolution、attempt context、next action
- 真实阻断场景不再需要 agent 额外猜测下一步
- installable template 真源与 managed hook parity 都被测试保护
- `.planning` 清晰记录这是 2026-05-12 的 special follow-up，而不是新 milestone
