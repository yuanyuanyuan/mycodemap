## 1. Repomix (最值得学习)

🔗 https://github.com/yamadashy/repomix

### 核心功能

```
✅ 多种输出格式: XML, Markdown, JSON, Plain Text
✅ 代码压缩 (--compress): 使用 Tree-sitter 提取签名，减少 70% Token
✅ MCP 服务器集成: 支持 Model Context Protocol
✅ 远程仓库支持: 直接处理 GitHub 仓库
✅ Token 计数: 精确统计文件和总 Token 数
✅ 文件分割: --split-output 1mb 分割大输出
✅ 安全扫描: 集成 Secretlint 检测敏感信息
✅ Git 历史: --include-logs 包含提交历史
✅ Git diff: --include-diffs 包含工作区变更
```

### 代码压缩功能（值得借鉴）

**Repomix 的压缩输出示例**:
```typescript
// 原始代码
import { ShoppingItem } from './shopping-item';

const calculateTotal = (items: ShoppingItem[]) => {
  let total = 0;
  for (const item of items) {
    total += item.price * item.quantity;
  }
  return total;
}

// 压缩后（保留结构，移除实现）
import { ShoppingItem } from './shopping-item';
⋮----
const calculateTotal = (items: ShoppingItem[]) => {
⋮----
```

**实现方式**:
- 使用 Tree-sitter 解析 AST
- 保留：imports, 函数签名, 类定义, 接口
- 移除：函数体实现, 注释, 空行
- 用 `⋮----` 标记省略的内容

### MCP (Model Context Protocol) 集成

**Repomix 提供的 MCP 工具**:
```typescript
1. pack_codebase        - 打包本地代码
2. pack_remote_repository - 打包远程仓库
3. grep_repomix_output   - 在输出中搜索
4. read_repomix_output   - 读取输出文件
5. file_system_read_file - 安全读取文件
6. file_system_read_directory - 列出目录
```

**使用场景**:
```bash
# AI 助手可以直接调用
"分析我的代码库"
→ AI 调用 pack_codebase
→ 获取上下文
→ 生成回答
```

## 3. Sourcegraph Cody (企业级)

🔗 https://sourcegraph.com/cody

### 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                     Cody Enterprise                          │
├─────────────────────────────────────────────────────────────┤
│  IDE 插件 (VS Code, JetBrains)                               │
├─────────────────────────────────────────────────────────────┤
│  Sourcegraph 平台                                            │
│  ├── Code Search (BM25 排名)                                 │
│  ├── Code Intelligence (代码图)                              │
│  └── Repository Indexing (仓库索引)                          │
├─────────────────────────────────────────────────────────────┤
│  LLM (Claude, GPT-4, Gemini)                                 │
└─────────────────────────────────────────────────────────────┘
```

### 核心技术

**1. RAG (Retrieval-Augmented Generation)**
```
用户提问 → 检索相关代码片段 → 构建 Prompt → LLM 生成回答
```

**2. BM25 排名算法**
- 类似搜索引擎的相关性排序
- 比简单的向量相似度更准确

**3. 代码图 (Code Graph)**
- 跨文件引用关系
- 符号定义和用法
- 继承和实现关系

**4. 长上下文窗口 (1M tokens)**
```
实验结果：
- Essential Recall: +15%
- Hallucination Rate: -40%
- 可以处理 4MB 以下的整个代码库
```

### 上下文分层架构

```
Permalayer (90%):      当前文件周围的代码
Action history (5%):   用户操作历史
Global retrieval (5%): 代码搜索结果
```

## 🎯 CodeMap 的差异化定位

### 当前优势

1. **双层文档系统**: AI_MAP.md + CONTEXT.md（Repomix 只有单个输出文件）
2. **符号级分析**: 函数签名、类成员、调用关系
3. **TypeScript 专注**: 深度解析 TS 类型系统
4. **Watch 模式**: 文件变更自动更新

### 需要追赶的功能

| 优先级 | 功能 | 参考实现 |
|--------|------|----------|
| P0 | 代码压缩 | Repomix --compress |
| P0 | Token 计数 | Repomix tiktoken |
| P1 | MCP 支持 | Repomix MCP tools |
| P1 | 安全扫描 | Repomix Secretlint |
| P2 | VS Code 扩展 | AI Context Generator |
| P2 | 浏览器扩展 | Gitingest extension |

### 独特机会

**1. 分层上下文策略**
```
CodeMap 已经实现：
- L1: AI_MAP.md (项目概览)
- L2: CONTEXT.md (模块详情)
- L3: 符号签名

可以扩展：
- L4: 关键代码片段
- 跨文件调用链
```

**2. 代码地图可视化**
```
Repomix 和 Gitingest 都没有的：
- Mermaid 依赖图
- 架构图生成
- 复杂度热力图
```

**3. 增量更新**
```
CodeMap 已有 Watch 模式
可以强化：
- 更快的增量解析
- 变更影响分析
```

## 📚 技术实现参考

### Repomix 的代码压缩实现

```typescript
// 使用 Tree-sitter 提取关键结构
import Parser from 'tree-sitter';
import TypeScript from 'tree-sitter-typescript';

const parser = new Parser();
parser.setLanguage(TypeScript);

function compressCode(source: string): string {
  const tree = parser.parse(source);
  const root = tree.rootNode;
  
  // 提取 imports
  const imports = root.descendantsOfType('import_statement');
  
  // 提取函数签名
  const functions = root.descendantsOfType('function_declaration');
  
  // 移除函数体
  for (const func of functions) {
    const body = func.childForFieldName('body');
    if (body) {
      replaceWithEllipsis(body);
    }
  }
  
  return generateOutput();
}
```

### Cody 的 BM25 排名

```python
# 简化的 BM25 实现
from rank_bm25 import BM25Okapi

def search_codebase(query: str, code_snippets: list):
    tokenized_corpus = [tokenize(doc) for doc in code_snippets]
    bm25 = BM25Okapi(tokenized_corpus)
    
    tokenized_query = tokenize(query)
    scores = bm25.get_scores(tokenized_query)
    
    return rank_documents(scores, code_snippets)
```
