## 📋 用户决策结果汇总

| 决策点 | 选择 | 结论 |
|--------|------|------|
| 1. CONTEXT.md 生成范围 | **A** | 每个 `.ts` 文件一个 CONTEXT.md |
| 2. CLI 命令命名规范 | **A** | 完全按文档实现，不添加简写别名 |
| 3. JavaScript 支持 | **A** | 仅支持 TypeScript，不支持纯 JS |
| 4. AI 功能优先级 | **A** | AI 生成功能是 MVP 核心功能 |
| 5. 配置文件格式 | **A** | 使用 JavaScript (codemap.config.js) |

---

### 决策详情与影响分析

#### 决策 1: CONTEXT.md 生成范围 = 每个 `.ts` 文件一个
**影响**: 
- ✅ 完整覆盖，每个文件都有上下文，AI 理解更全面
- ⚠️ 大型项目（5000+ 文件）将生成 5000+ CONTEXT.md 文件，占用存储空间大
- 💡 建议: 实现文件数阈值警告，超过 1000 文件时提示用户

#### 决策 2: CLI 命令命名规范 = 完全按文档实现
**影响**:
- ✅ 命令语义清晰，符合使用指南文档
- ✅ 用户文档和实际体验一致
- ⚠️ 命令较长，需要更多输入（后续可考虑 v2.0 添加别名）

#### 决策 3: JavaScript 支持 = 仅 TypeScript
**影响**:
- ✅ 简化解析器实现，专注 TS AST
- ✅ 可以充分利用 Type Checker 获取类型信息
- ⚠️ 不支持 JSDoc 类型的纯 JS 项目

#### 决策 4: AI 功能优先级 = 是 MVP 核心功能 ⭐
**影响**:
- ✅ AI_MAP.md 包含 AI 生成的项目概述，提升产品价值
- ⚠️ 需要集成 OpenAI/Anthropic API，增加外部依赖
- ⚠️ 需要处理 API 调用失败时的降级逻辑
- 💡 **关键实现点**: 
  - 必须实现 Token 预算管理
  - 必须实现 AI Provider 抽象层
  - 必须提供离线模式（仅生成结构化数据）

#### 决策 5: 配置文件格式 = JavaScript
**影响**:
- ✅ 支持注释和条件逻辑，配置更灵活
- ✅ 与设计文档中的示例一致
- ⚠️ 配置文件可被审查工具标记（代码执行风险）
