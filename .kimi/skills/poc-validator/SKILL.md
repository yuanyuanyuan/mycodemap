# POC Validator — 真实场景可行性验证技能

> **一句话**：POC 不是"跑通就行"，而是证明**真实场景下**确实可行、可恢复、可度量。

## 触发词（速查）

| 用户说 | 激活等级 |
|--------|---------|
| "做个 POC" / "验证一下可行性" / "确认能跑通" | 必须激活 |
| 涉及 Stripe / OpenAI / AWS / 支付网关 / MCP server | 必须激活 |
| 涉及"子代理规则注入/检索/行为改变" | 必须激活 |
| 涉及 CLI → 文件 → 输出 / 前端 → 后端 → 数据库 | 必须激活 |
| 涉及"大数据量 / 高并发 / 实时计算" | 必须激活 |
| "用 mock 测了很快" / "单元测试过了" | **必须纠正**：mock ≠ 真实可行 |

---

## 决策树：3 步判定执行路径

```
用户请求
  │
  ├─ 涉及第三方/内部 API 调用？ ──→ 类型 A：API 集成型
  ├─ 涉及子代理/规则/上下文传递？ ──→ 类型 B：子代理行为型
  ├─ 涉及 CLI/文件管道/构建流程？ ──→ 类型 C：数据流/端到端型
  └─ 涉及大数据/高并发/算法性能？ ──→ 类型 D：性能型

所有类型继续向下：
  │
  ├─ 时间 < 30min，风险低，有 sandbox ──→ 快速路径（5 步精简）
  └─ 时间充裕，风险高，无明确 sandbox ──→ 深度路径（6 步完整 + 检查点）
```

**风险信号（出现任一必须用深度路径）**：
- 涉及资金/支付/生产数据
- 可能触发 rate limit / 账号封禁
- 子代理有写权限或能执行破坏性命令
- 无现成的 test/staging 环境

---

## 🔒 强制检查点（所有路径通用）

**未通过以下确认，不得执行任何真实调用。**

| 检查点 | 确认问题 | 若否 |
|--------|---------|------|
| CP-0 环境 | 是否有 sandbox / test 环境？API key 是测试凭证？ | 终止 POC |
| CP-1 数据 | 真实数据是否已脱敏？有使用授权？ | 改用合成数据 |
| CP-2 破坏 | 失败测试是否可能触发告警/限流/封号？ | 缩小失败范围 |

---

## 快速路径（≤30min，5 步）

适用：风险低、有现成 sandbox、用户急需快速结论。

| 步骤 | 动作 | 输出物 | 时间盒 |
|------|------|--------|--------|
| R1 隔离 | `cp -r project /tmp/poc-$(date +%s)` | 隔离目录路径 | 2min |
| R2 Happy Path | 直接调用目标功能一次 | 输出截图 / 日志片段 | 5min |
| R3 真实数据 | 替换 mock，用真实请求/文件/Agent 跑一次 | 请求/响应记录 | 10min |
| R4 一次失败 | 故意触发一个错误（错误 key / 不存在文件 / 超大输入） | 错误截图 + 进程状态 | 5min |
| R5 结论 | 填写「快速验证结论表」（见附录 A.3） | 结论表 | 8min |

**快速路径通过标准**：Happy Path 成功 + 失败场景不 crash + 有真实调用证据。

## 深度路径（完整 6 步 + 检查点）

适用：风险高、需要说服利益相关者、或快速路径未通过时。

### Step 1: 环境隔离（Replicate）

```bash
cp -r /data/project /tmp/project-poc
# 构建、数据库、API key 全部在隔离副本中配置
```

- 数据库 → docker 临时实例
- API key → test/sandbox key（**禁止生产 key**）
- 构建 → 隔离副本内 `npm run build`

### Step 2: 功能可用（Function）

直接调用目标功能，验证最基础 happy path：

```bash
node dist/cli/index.js <new-command>
curl -s http://localhost:3000/api/health
```

**判定标准**：exit 0 + 输出可解析 + 符合数据模型。

### Step 3: 真实数据（Real Data）

**替换所有 mock**：
- API 型：真实 HTTP 请求替代 `jest.mock('axios')`
- 数据流型：真实 1000+ 行文件替代 3 条 fixture
- 子代理型：真实 `Agent` spawn 替代假想的"子代理会这样做"

**验证记录**：保留真实请求/响应日志（脱敏）、子代理 tool call 记录、文件处理前后 diff。

### Step 4: 失败场景（Failure）

**必须验证至少一个真实失败路径**：

| POC 类型 | 推荐失败场景 |
|----------|-------------|
| API 集成 | 错误 credential → 401；非法参数 → 400；超时 → 明确错误 |
| 子代理行为 | 规则缺失 → 拒绝盲目操作；工具调用失败 → 报告失败 |
| 数据流 | 文件不存在 → 清晰错误；中间步骤 crash → 无残留临时文件 |
| 性能 | 数据量翻倍 → latency 曲线；并发超限 → 无 5xx 或死锁 |

**判定标准**：失败时有明确错误信息（不 hang、不吞异常）+ 失败后状态可预期（无脏数据、无残留）。

### Step 5: 对照验证（Baseline）

**目的**：证明新方案有增量价值。

| 对照组 | 实验组 | 度量指标 |
|--------|--------|---------|
| 旧 SDK / 旧流程 | 新 SDK / 新流程 | latency、错误率、内存占用 |
| 无检索指引 | 有检索指引 | 规则覆盖数、违规操作数 |
| 单线程 / 无缓存 | 并行 / 有缓存 | 吞吐量、CPU 利用率 |

**没有对照组 = 无法证明 POC 价值。**

### Step 6: 证据汇总（Evidence）

必须输出：

```markdown
## 实验结果汇总

| 实验 | 假设 | 验证方式 | 结果 | 状态 |
|------|------|---------|------|------|
| A: 功能可用 | ... | ... | ... | ✅/❌ |
| B: 真实数据 | ... | ... | ... | ✅/❌ |
| C: 失败场景 | ... | ... | ... | ✅/❌ |
| D: 对照验证 | ... | ... | ... | ✅/❌ |

## 关键证据
- [证据] ...

## 可信度自评
- **确定信息**：...
- **推测信息**：...
- **需验证信息**：...
- **风险标记**：...
```

## 四真原则检查清单

| # | 原则 | 快速验证法 | 未通过信号 |
|---|------|-----------|-----------|
| 1 | 真实环境 | 请求域名含 sandbox/staging /tmp 非源码目录 | 用了 localhost mock server |
| 2 | 真实数据 | 数据量级 ≥ 1000 条 / 请求参数非硬编码 fixture | 只有 3 条测试数据 |
| 3 | 真实调用 | 进程列表可见真实子进程 / 网络监控可见真实出站请求 | 只断言了函数返回值 |
| 4 | 真实失败 | 错误日志真实存在，非人工构造的 throw | 没有失败场景记录 |

---

## 常见反模式（必须避免）

| 反模式 | 为什么错 | 快速识别法 |
|--------|---------|-----------|
| 纯 mock 验证 | 掩盖序列化、时区、编码问题 | 代码中出现 `jest.mock` / `nock` |
| 只看 happy path | 没验证错误 credential、超时、rate limit | 实验记录中只有 ✅ 无 ❌ |
| 数据量太小 | 无法发现内存溢出、特殊字符问题 | 测试数据 < 100 条 |
| 无对照组 | 无法证明新方案有优势 | 没有"旧方案 vs 新方案"对比 |
| 源码目录直接改 | 污染工作树，可能触发 hooks | 操作路径不含 `/tmp` 或隔离目录 |
| 观察指标模糊 | "看起来快了"无法量化 | 结果中没有 ms / MB / % 数字 |

## 验收清单（DoD）

- [ ] **源码未污染**：所有修改在 `/tmp` 隔离环境
- [ ] **真实环境**：使用了 sandbox / staging / 隔离实例
- [ ] **真实数据**：用了真实请求/文件/记录
- [ ] **真实调用**：用了真实子进程/网络请求/Agent spawn
- [ ] **失败场景**：至少验证了一个真实失败路径
- [ ] **对照验证**：有对比实验（新 vs 旧、有 vs 无）
- [ ] **可判定标准**：每个实验有明确的通过/不通过判定依据
- [ ] **证据留存**：保留了真实请求日志、响应数据、tool call 记录（脱敏后）
- [ ] **临时文件已清理**：`rm -rf /tmp/project-poc`

## 技能元数据

- **适用范围**：API 集成、AI 代理工作流、CLI 工具链、数据流管道、性能基准
- **不适用**：纯算法数学证明、纯 UI 视觉走查、已上线功能的回归测试
- **前置依赖**：被验证对象可构建/可部署、有 sandbox 或隔离环境可用
- **预期耗时**：快速路径 15-30min / 深度路径 60-90min
