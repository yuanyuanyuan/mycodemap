# docs/exec-plans/

本目录存放执行计划与落地过程，不存放长期稳定规则。

## 当前结构

- `active/`：当前活跃计划
- `completed/`：已完成或已归档的历史计划
- `tech-debt/`：技术债、待处理项、批量修复清单

## 当前已迁入

## 写作边界

- 计划要写目标、限制、DoD、依赖、风险。
- 复盘要写实际结果、偏差、残留问题。
- 技术债要可排序、可追踪、可关闭。
