# 任务计划（根据 review 建议修订文档）

## 任务目标
- 按 review 结论修订重构相关文档，消除关键契约冲突，保证可实施一致性。

## 约束条件
- 只修改文档，不改代码实现。
- 所有修改需与“单一真源”原则一致（特别是风险评分公式）。
- 保留现有文档结构，尽量做最小侵入修订。

## 验收标准
- 工作流持久化、状态恢复、阶段推进描述自洽。
- workflow phase 与 analyze/ci 命令契约一致。
- 风险评分公式在需求/模块/CI 三处完全一致。
- 输出契约校验与 analyze machine/json 输出协议一致。
- TestLinker 源文件推断规则与示例一致。
- rg 可见性策略在所有文档中不冲突（仅内部兜底）。

## 执行阶段
- [in_progress] Phase 1: 修订需求/架构总览中的全局契约
- [pending] Phase 2: 修订模块详细设计（编排器/Git/融合/测试关联/CI）
- [pending] Phase 3: 交叉复核并记录变更摘要

## 失败模式预演
- 失败案例：只改单文档会再次出现“公式不一致”或“命令契约冲突”，导致实现阶段无法落地。
- 预防：每次改动后做跨文档关键字复核（公式、intent、workflow 命令、json 输出模式）。
